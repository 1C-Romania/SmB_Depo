Designed to view data synchronization statistics.

You can see results of data synchronization as a link in the Data synchronization list, e.g. in the application desktop. According to the results, the link looks like No warnings (data has been synchronized successfully) or  Warnings (number of warnings within brackets). To open the synchronization result list, click the link. 

Data synchronization might require user intervention. The list columns contain necessary information about the irregular data why it was rejected. The cause is briefly stated in a special column. Highlighted string in the lower part of the list gives a detailed wording specifying problem list/document fields.

Filters

Simplify preview with the filters:

  •  Select period of data presentation
  • In the Sender field, Select an infobase you need from the list.
  • In the Cause field, write an error cause (this filter is not available for the Conflicts tab).

Problem situations may be divided into the following categories that are displayed on separate tabs in the list. The number within brackets in the title of each tab stands for the number of specific problem situations:

  • Unposted documents

  • This list shows documents that for some reason were not posted. To solve the problem of an unposted document, open the document and correct the errors preventing its posting (e.g. an insufficient product balance). At the bottom of the list, you can see why automatic posting failed.
  • Click Edit selected to try to correct the errors in the selected documents. 
  • When all the errors are corrected, you can Post the document. 
  • Click  More - Edit to open a document.
  • Click Ignore to hide a document without solving the problem.
  • Click More - Show ignored to show ignored documents.
  • To return an ignored document, click More - Do not ignore.

Uncompleted fields

  • Contains a list of uncompleted catalog item fields. To correct them, open the catalog item (double-click it) and fill in the required fields. Then save the item.
  • Click Edit selected to try to fill in the fields in the selected items.
  • Click More - Edit  to open an item.
  • Click Ignore to delete an item from the list without filling in the required fields. 
  • Click More - Show ignored to show ignored items.
  • To return an ignored item to the list, click More - Do not ignore.

Conflicts

  • When data is synchronized between two applications, the same data may sometimes be edited simultaneously (between synchronization sessions) in both applications. It results in two versions of the same data in the applications. This situation is known as a data change conflict. Such conflicts are resolved (by selecting one version) automatically. To change it, select a list item.
  • In a row at the list bottom, you can see a method the application selected.
  • To see differences between two versions, click Show differences.
  • To select one version, you can view each. Click More - Show version from this infobase or More - Show version from another infobase.
  • To view a list item, click More - Change data.
  • After you select the version to save, click Confirm to save the result of automatic conflict resolution or Review to select the opposite result. After that, the item gets inactive, gets the  Reviewed flag and, by default, is not shown in the list. 
  • By default, the reviewed conflicts are hidden. To show them in the list, click More - Show reviewed conflicts.
  • To check the reviewed conflicts again and change something, click More - Clear Reviewed flag.

Data of rejected date

  • This list shows the data rejected because it was forbidden to load data of previous periods.
  • To compare rejected data with the current data, click Show differences (if the current application contains data).
  • Click More - Change to open an item.
  • Click More - Show rejected version to view another version.
  • Click Accept version to accept data despite the data loading prohibition.
  • Click Ignore to delete a line from the list. 
  • Click More - Show ignored to view ignored items.
  • Click More - Do not ignore to return an ignored item to the list.
