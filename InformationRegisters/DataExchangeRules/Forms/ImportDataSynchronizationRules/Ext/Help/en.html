It is intended for control and replacement of the sets of rules for data synchronization.

It is opened by clicking the Import the set of rules option from data synchronization settings.

By default, data synchronization is performed in accordance with so called Standard exchange rules. Exchange rules consist of object conversion rules and object registration rules.

You can open the Object conversion rules from the data synchronization settings using the More - Import object conversion rules command. Data conversion rules are used in synchronization of data of two applications.

You can open the Object registration rules from the data synchronization settings by the More - Import object registration rules command.

Rules source

  • For standard synchronization of data between standard configurations, the From configuration standard rules are used. However, in some cases it is necessary to use the From file on computer rule (for example if the synchronization is performed between bases with non-standard configurations)
  • Using the option button you can specify where to take object registration rules from:
    • From configuration (selected by default)
    • From file on computer
  • Press Import, specify the file path. Use of rules, imported from a file, may lead to errors when updating to a new version of the application
  • The names of the rules currently in use are displayed below

Information on set of rules

For the purpose of control, the application displays the following information on rules:

  • A warning that errors might occur when using rules from an external source
  • Object registration rules of this infobase - the versions and dates of creation of the rules are displayed;
  • Object conversion rules for this infobase - the versions and dates of creation of the rules are displayed;
  • Rules of correspondent conversion - the versions and dates of creation of the rules are displayed

Debug mode

  • If the From file on computer option is selected, the debug mode becomes available
  • Click the Debug mode adjustment link.
  • Select the Activate debug mode checkbox. After this, the following settings become available:
    • Connect external data processor for export debugging – the external data processor file is specified that contains the export data processors debug module generated in the Data conversion configuration. IN the client-server mode of operation, the path on the server is specified
    • Connect external data processor for import debugging – the external data processor file is specified that contains the import data processors debug module generated in the Data conversion configuration. IN the client-server mode of operation, the path on the server is specified
    • Mode of data exchange logging – if enabled, events and errors that occur in the process of data exchange will be recorded in the specified file. If the file is not specified, the messages will be displayed in the event log
    • Not to stop data exchange in case of error – if set, execution of data synchronization will not be stopped when an error occurs. Exchange of data will be completed with displaying debugging messages in the exchange protocol file or event log

Import of rules

  • Enable the option button for From file on computer
  • Press Import
  • Specify the path to the file with the rules

Export of rules

  • When needed, registration rules can be exported to a file
  • To do that, click More - Save rules in file. Select the path to the file on the computer or on a network drive, the name is assigned by the application by default. Click Save

Replacement of standard object conversion rules by rules from external File

  • Select From file on computer and using the Import option select the rules file on the computer or in a network folder
  • Pay attention to the fact that the correct rules suitable for the used versions of the source application and receiver application are imported. For this purpose Information on set of rules is displayed with dates of the rules
  • Click Save and close. The imported rules will be applied immediately

Note: object conversion and registration rules are used for all similar data synchronizations (that are based on the same exchange plan; for example: two data synchronizations are set based on the Full exchange plan: Synchronization with Krasnodar branch and Synchronization with Kiev representative office). Therefore if several similar synchronizations are set in the application, then it would be sufficient to replace rules only once (by selecting any of the adjusted data synchronizations).
