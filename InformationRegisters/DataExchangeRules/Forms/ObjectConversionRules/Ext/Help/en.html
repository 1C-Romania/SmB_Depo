Designed to control and edit data conversion rules. Data conversion rules are used to synchronize data of two applications. 

Default settings of data synchronization comply with the Standard exchange rules. The exchange rules consist of object conversion rules and object registration rules.

Click More - Import object conversion rules to open the object conversion rules in the data synchronization setting.

Rules source

  • Standard data synchronization between standard configurations uses standard rules of the configuration (identifying a name of the conversion rules template providing the standard rules).  However, sometimes you need to use rules from a file (e.g. when you synchronize bases with non-standard configurations).

  • Select where to get object registration rules:

  • Use standard rules (by default).
  • Import from file on computer. Rules imported from a file may cause errors when you start using a new application version.
  • Below are the names of the rules being used.

Rules data

  • Monitor information with the data displayed:
    • Warning about possible errors when using rules from an external file.
    • Conversion rules for this infobase - the object conversion rules version and the rules creation date.
    • Correspondent conversion rules - the object conversion rules version and the rules creation date.

Debugging mode 

  • If the Import from file on computer variant is selected, you can use debugging mode.
  • To work with this mode, click the Enable debugging mode link,
  • Select the Enable debugging mode check box.
  • You get access to the following settings:
    • Connect external processing for export debugging – specify a file of external processing containing a export debugging module generated in the Data conversion configuration. IN client-server operation mode, specify a path on server.
    • Connect external processing for import debugging – specify a file of external processing containing an import debugging module generated in the Data conversion configuration. IN client-server operation mode, specify a path on the server.
    • Data synchronization log mode– if enabled, events and errors occurring during data exchange will be saved in the specified file. If the file is not specified, messages will be displayed in the Registration log.
    • Do not stop date exchange in case of error – if enabled, data synchronization will not stop even after an error occurs. Data exchange will be completed with debugging messages displayed in the exchange protocol or registration log.

Importing rules

  • Select Import from file on computer.
  • Click Import.
  • If necessary, specify a path to the file with rules.

Exporting rules

  • If necessary, you can export registration rules to a file. Click Save conversion rules to file.
  • Select a path to the file on the computer or network drive. Name will be selected automatically.  Click Save.

Replacing standard object conversion rules with rules from external file

  • Select Import from file on computer, click Import and select the rules file on the computer or in the network folder.
  • Make sure that you are importing correct rules, compatible with the current versions of the source application and receiver application.  Check the Rules information with the date of your rules. 
  • Click Save and close. The imported rules will be applied immediately.

Note: object conversion rules and object registration rules are used for all same-type data synchronizations (based on the same exchange scheme; e.g. the Complete exchange scheme is the basis of two data synchronizations: Synchronization with Chicago office and Synchronization with Atlanta office). Therefore if there are multiple same-type synchronizations, you only have to change rules once (by selecting any of the configured data synchronizations).
