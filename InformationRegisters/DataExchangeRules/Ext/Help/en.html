By default, data synchronization is performed in accordance with so called Standard exchange rules. If these rules are out of date, execution of data synchronization would lead to very serious errors and damage of information. Therefore, when setting a new data synchronization, and also when a new update of any application involved in data synchronization is released, relevancy of the standard exchange rules shall be verified.

Standard exchange rules consist of object conversion rules and object registration rules. Data conversion rules are used in synchronization of data of two applications. Registration rules are provided for all types of data synchronization. The set of relevant exchange rules is published on the technical support site http://users.v8.1c.en. Correctional versions of exchange rules are also published on this site.

Verification of relevancy of synchronization rules

  • When verifying relevancy of the rules, the attention shall be paid to the following controlled parameters:
    • Release numbers of the source application and the receiver application for which the rules are intended
    • Date of rules creation

  • The verification should be performed according to the following rules:

    • values of controlled parameters imported in the source application are determined
    • values of controlled parameters imported in the receiver application are determined
    • values of controlled parameters included in the source application delivery package are determined
    • values of controlled parameters included in the receiver application delivery package are determined
  • After analysis of all these rules, rules with the most current dates of creation are selected. When analyzing the rules, configuration release numbers shall be taken into account. Configuration release numbers specified in the rules shall correspond to the configuration releases, between which data synchronization is being adjusted

Verification of rules imported to applications

  • Create a new data synchronization, but do not start it
  • Open the created data synchronization setting
  • On the More menu of the data synchronization setting, run the Import object conversion rules command
  • Check information on the data conversion rules
  • Then run the More - Import object registration rules command
  • Check information on the object registration rules
  • You can analyze the controlled parameters imported in the application for already existing data synchronization setting in a similar way. For example, to check the object registration rules in the existing data synchronization setting in a slave node, use the Import object registration rules command
  • To verify data synchronization rules for two applications, these actions should be made successively for the source application and for the receiver application

Verification of rules supplied with applications

  • In the Data conversion rules window (Data registration rules), select the From file on computer option using the option button. Execute the Import command
  • Select the conversion rules file in the application delivery package
  • The file path depends on the application delivery package. It is usually located in the Data exchange folder
  • Verify information about the rules
  • These actions should be made successively for both applications involved in the data synchronization

Control of rules from external file

  • It is necessary to read information about these rules, then to save a copy of the rules used in a file using the Save conversion (registration) rules in file option
  • Select the Use standard rules option 
  • View the displayed information on the rules

Making decisions based on verification results

  • As a result of rules verification, the newer version is chosen and one of the following decisions is taken:
  • Continue using the current rules
    • Additional actions are not required
  • Use the latest rules from the set of rules in one of the applications
    • Order of importing is described in the file which is usually named as Exchange rules connection.txt, you can also find it in the Data exchanges directory
  • Use standard object conversion and registration rules built in the configuration instead of rules from an external source
    • To do this, it is necessary to run the More - Export object conversion rules (object registration rules) command. Then select the Use standard rules option
