It is used to customize object versioning settings.

Using versioning of documents and catalogs you can track change history.

To log object change history you shall select the Common settings command of Administration section, select the Versioning objects check box (selected by default).

After this the Versioning objects command used to customize settings is available in the Administration - General settings sections.

The list displays:

  • Object - application object list in a tree form with object type grouping
  • Versioning option - selected versioning option

Selecting versioning option

Select one or several objects, run the Select versioning option command, then select the required versioning option for each type of documents and catalogs from the drop-down list. You can also find the command in the More menu or in a shortcut menu by right-clicking.

  • No versioning - history of object versions is not written
  • Versioning when writing - new record is entered in version history each time when changing (or creating) catalog or document
  • Versioning when posting - versions are saved only during posting process; applicable only for documents with posting allowed
  • By default - Set recommended settings (for catalogs - no versioning, for documents - versioning when posting)

It is also possible to configure versioning for the whole group of application objects, for example, you can select all documents or catalogs.

Using versioning

  • At the beginning of the application implementation, when data size is small enough and responsible users make a lot of errors in the working process (for example, they enter excess information or delete necessary information accidentally), versioning can help to arrange the lists and documents, delete mistakes/errors, help users to understand their errors and eliminate such errors/mistakes in future.
  • With the help of object versioning you can select any previous version of the document or catalog item being more detailed and correct comparing with the last version.
  • Later, when information volume increases, you can gradually refuse from versioning of some objects or apply versioning only at critical moments, for example, when posting documents.
  • Versioning large number of objects can lead to size increase of data saved in the application because of the versions saved, and this can significantly slow down the application functioning. Therefore, it is recommended to use this option selectively.

Versioning when posting mode features

  • If the Versioning when posting mode is selected for the document, then the first version of the document will be written in the history only after the first posting of the document. Document versions will be written later at each posted document record. The mode excludes creating versions of new documents and documents not filled completely in order not to increase the size of data Saved in the application. This setting is recommended for all documents.
