Electronic documents package document

The document is formed by the application automatically, and is intended to store sender's and recipient's electronic documents attributes, as well as to link the electronic documents with registration system documents.

For outgoing documents the data of the Electronic document and Owner's object is filled at the moment of package creating, for incoming documents the data is filled at the moment of unpacking which is performed either automatically by the application or by running the Unpack package command (the package's status shall be "To unpack").

In certain cases the document can be used for analysis of problem situations by technical support service. To do this, it is necessary to load the package content by running the Save package on disk command.
