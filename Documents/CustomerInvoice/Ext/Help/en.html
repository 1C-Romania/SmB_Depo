Customer invoice document

The document registers sales of inventory to customers in the database.

Document transaction types:

  • Customer Sale.

  • Transfer to Consignment.

  • Transfer to processing.

  • Transfer for Safe Custody.

  • Return to vendor.

  • Return to principal.

  • Return from processing.

  • Return from Safe Custody.

Document attributes:

  • Company - company, for which the document is Creatingd. It is selected from the Companies list. The attribute is displayed on the form if Accounting by Multiple Companies is enabled in accounting options.
  • Transaction kind - of the document.
  • Counterparty - customer selected from the Counterparties list.

  • Company contract - agreement between company and company. It is selected from the list of Counterparty Company contracts.

  • Basis - one of the following documents: Supplier invoice or Invoice For payments. The attribute is mandatory for the transactions of type Return to vendor, Return to principal or Return from Safe Custody.

  • Order - of a sale transaction. Document Customer order is selected for the following types of transactions: Customer Sale, Consignment or Return from processing. Document Purchase order is selected for the following transaction types: Return to vendor, Return to principal or Transfer to processing. The attribute is displayed on the form if order is enabled for the entire document in the accounting options. The attribute is optional. Note: the attribute is not displayed on the form for the transaction types of Transfer for Safe Custody or Return from Safe Custody.

  • Structural unit, warehouse - warehouse which is selected from the Structural units list from items of the Structural unit, warehouse type.

  • Cell - cell of the warehouse from which the inventory is being WrittenOff. It is selected from the Structural unit, warehouse Cells list. The attribute is displayed on the form if warehouses having cells are enabled in the accounting options and warehouse does not have the order flag.

To change the price types, discounts and currencies, click on the Edit Prices and Currencies button.

  • Inventory and toolss. Document table can be filled using pick method (button Pick on the command bar), or data can be entered manually. Table fields:

    • ProductsAndServices - item, being sold, selected from the ProductsAndServices list with the Inventory item type.

    • Characteristic - if necessary, characteristic of the item may be selected in this field. It is selected from the Products and services characteristics list. The attribute is displayed on the form if characteristics are enabled for the items in accounting options.

    • Batch - if necessary, batch of the item may be selected in this field. It is selected from the Products and services batches list. The attribute is displayed on the form if batches are enabled for the items in accounting options.

    • Quantity - COUNT of items sold.

    • Reserve - COUNT of items in reserve.

    • UOM - unit of measure. It is selected from the UOM Classifier list or from the Units of Measure list.

    • Price - price of item sold.

    • Discount, % - discount (markup) rate.

    • Amount - cost of item sold.

    • VAT rate - VAT rate selected from the VAT rates list.

    • Including VAT - Amount of VAT.

    • Total - total Amount calculated according to the price and currency settings. To change the settings click on the Edit Prices and Currencies button.

    • Order - of a sale transaction. Document Customer order is selected for the following types of transactions: Customer Sale, Consignment or Return from processing. Document Purchase order is selected for the following transaction types: Return to vendor, Return to principal or Transfer to processing. The attribute is displayed on the form if order is enabled for each document line in the accounting options. The attribute is optional. Note: the attribute is not displayed on the form for the transaction types of Transfer for Safe Custody or Return from Safe Custody.
    • Content - text of the item in the print form. To enable the attribute go to All actions - Change form... and check the flag.

  • Setoff of advance payment. The table contains prepayments received from customers earlier. To edit data in Prepayment table, click on the Edit Setoff of advance payment button. Table fields:

    • Document - document with customer prepayment: Petty Banking Receipt, Payment receipt.

    • Amount of settlements - Amount in contract currency.

    • Rate - rate used to recalculate Amount to document currency.

    • Factor - factor used to recalculate Amount to document currency.

    • Payments Amount - Amount of prepayment in document currency.

  • Additional datarmation. Belongs to group of attributes:

    • Structural unit - division, responsible for transaction, which is selected from the Structural units list from items of the Structural unit type.

    • Responsible - information about the employee who enters current document. It is selected from the Performers list. The attribute is optional.

    • Consignee - company to which items will be shipped. It is selected from the Counterparties list.

    • Counterparty - company which ships items to consignee. It is selected from the Counterparties list.

    • Note - any additional information.

The Customer invoice document can be entered on the basis of the following documents:

  • Supplier invoice.
  • Invoice for Payments.
  • Customer Order.
  • Work order.
