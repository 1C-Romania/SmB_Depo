Supplier invoice document

The document registers purchases of inventory from vendors in the database.

Document transaction types:

  • Receipt from vendor.

  • Receipt to Consignment.

  • Receipt to processing.

  • Receipt for Safe Custody.

  • Return from customer.

  • Return from consignee.

  • Return from the processing company.

  • Return from Safe Custody.

Document attributes:

  • Company - company, for which the document is Creatingd. It is selected from the Companies list. The attribute is displayed on the form if Accounting by Multiple Companies is enabled in accounting options.
  • Transaction kind - of the document.
  • Counterparty - vendor selected from the Counerparties list.

  • Company contract - agreement between company and company. It is selected from the list of Counterparty Company contracts.

  • Basis - document Customer invoice. The attribute is mandatory for the transaction type Return from customer.

  • Order - of a purchase transaction. Document Purchase order is selected for the following transaction types: Receipt from vendor, Receipt to Consignment or Return from the processing company. Document Customer order is selected for the following types of transactions: Return from customer, Return from consignee or Receipt to processing. The attribute is displayed on the form if order is enabled for the entire document in the accounting options. The attribute is optional. Note: the attribute is not displayed on the form for the transaction types of Transfer for Safe Custody or Return from Safe Custody.

    Structural unit, warehouse - warehouse which is selected from the Structural units list from items of the Structural unit, warehouse type.

  • Cell - cell of the warehouse to which the inventory is being received. It is selected from the Structural unit, warehouse Cells list. The attribute is displayed on the form if warehouses having cells are enabled in the accounting options and warehouse does not have the order flag.

  • Include Expenses in Cost Price - attribute is displayed on the form for the Receipt from vendor transaction type .

To change the price types, discounts and currencies, click on the Edit Prices and Currencies button. IN case when the Register Vendor Prices flag is checked, prices will be recorded in register for future analysis.

  • Inventory. The table can be filled by using the Pick method (command bar button Pick of the table) or the manual data input. Attributes of the table:

    • ProductsAndServices - being purchased, selected from the ProductsAndServices list with the Inventory item type.

    • Characteristic - if necessary, characteristic of the item may be selected in this field. It is selected from the Products and services characteristics list. The attribute is displayed on the form if characteristics are enabled for the items in accounting options.

    • Batch - if necessary, batch of the item may be selected in this field. It is selected from the Products and services batches list. The attribute is displayed on the form if batches are enabled for the items in accounting options.

    • Quantity - COUNT of purchased items.

    • UOM - unit of measure. It is selected from the UOM Classifier list or from the Units of Measure list.

    • Price - price of purchased item.

    • Amount - cost of purchased item.

      VAT rate - VAT rate selected from the VAT rates list.

    • Including VAT - Amount of VAT.

    • Total - total Amount calculated according to the price and currency settings. To change the settings click on the Edit Prices and Currencies button.

    • Cost of Goods Sold - cost price of the returned item. The attribute is displayed on the form for the Return from customer transaction type .

    • Order - of a purchase transaction. Document Purchase order is selected for the following transaction types: Receipt from vendor, Receipt to Consignment or Return from the processing company. Document Customer order is selected for the following types of transactions: Return from customer, Return from consignee or Receipt to processing. The attribute is displayed on the form if order is enabled for each document line in the accounting options. The attribute is optional. Note: the attribute is not displayed on the form for the transaction types of Transfer for Safe Custody or Return from Safe Custody..

    • Amount of expense - Amount of expenses distributed on the cost of purchased items. The field can be filled automatically via the command bar button Distribute of the table Inventory. The attribute is displayed on the form if for the Receipt from vendor transaction type .

    • Content - text of the item in the print form. To enable the attribute go to All actions - Change form... and check the flag.

  • Expenses. Document table can be filled using pick method (button Pick on the command bar), or data can be entered manually. The table is displayed on the form for the Receipt from vendor transaction type. Table fields:

    • ProductsAndServices - item, which cost is being distributed on the inventory items, selected from the ProductsAndServices list with the Expense item type.

    • Quantity - COUNT of items.

    • UOM - unit of measure. It is selected from the UOM Classifier list or from the Units of Measure list.

    • Price - item price.

    • Amount - item cost.

    • VAT rate - VAT rate selected from the VAT rates list.

    • Including VAT - Amount of VAT.

    • Total - total Amount calculated according to the price and currency settings. To change the settings click on the Edit Prices and Currencies button.

    • Work order - related to expenses. It is selected from the Work orders list. The attribute is optional.

    • Customer Order - where expenses will be attributed. It is selected from the Customer order list. The attribute is displayed on the form if the flag Include Expenses in Cost Price is checked.

    • Structural unit - where expenses will be attributed. It is selected from the Structural units list with the Structural unit type. The attribute is displayed on the form if the flag Include Expenses in Cost Price is checked.

    • Business Activity - of the company where expenses will be attributed. It is selected from the Business Activities list. The attribute is displayed on the form if the flag Include Expenses in Cost Price is checked.

    • Content - expense item text in the print form. To enable the attribute go to All actions - Change form... and check the flag.

  • Prepayments. The table contains prepayments paid to vendors or received from customers earlier. To edit data in Prepayment table, click on the Edit Setoff of advance payment button. Table fields:

    • Document - document with prepayment: Expense, Expense from Petty Banking, Receipt to Banking, Payment receipt.

      Amount of settlements - Amount in contract currency.

    • Rate - rate used to recalculate Amount to document currency.

    • Factor - factor used to recalculate Amount to document currency.

    • Payments Amount - Amount of prepayment in document currency.

  • Additional datarmation. Belongs to group of attributes:

    • Note - any additional information.

The document Supplier invoice can be entered on the basis of the following documents:

  • Customer invoice.
  • Customer Order.
  • Work order.
