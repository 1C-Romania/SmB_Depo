Event document

With this document, users send email and register events.

 

When you create a new event, you need to specify its type. The type determines the data to fill in.

 

The Email message event type requires the following fields:

  • Contacts. This tabular section contains a list of persons to which you send the message. The tabular section includes:

    • Contact - an addresse. You can fill it with a line or an item of the Counterparties and Contact persons catalogs. When you click the selection button, an address book form opens.

    • Email address - the address of the sending. You can specify multiple addresses separated with [;].

  • Attachments. This tabular section includes a list of attachments to send. Add new attachments with the context menu or drag files from the OS. When you send the email, its attachments are saved as attached files.

  • Subject. Message subject.

  • Content. Message body.

  • Account. Email account from which the message will be sent.

  • State. If the email is sent successfully, the state is "Complete".

For other types, fill in the following fields:

  • State. This required field contains the event classification category. Fill in the field with the Event state catalog.

  • Counterparty. This optional field contains information about the counterparty. Fill in the field with the Counterparties catalog.

  • Subject. This optional field contains a short description of the event.

  • Content. This optional field contains a description of the event.

  • Underlying document. This optional field contains the documentary authority of the event.

  • Event start. This optional field contains the date/time of the event start.

  • Event end. This optional field contains the date/time of the event end.

  • Responsible person. This optional field contains information about the employee responsible for adding the current document. Fill in the field with the Employees catalog.

  • Participants. This tabular section contains a list of the event participants and includes the following fields:

    • Contact - a meeting participant. Fill in the field with the line or the Counterparties, Contact persons catalogs.

    • Contact information - contact details of a participant.

 

The Additional fields can include more fields defined by the user.
