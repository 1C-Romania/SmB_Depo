Designed to manage external user groups.

Groups are useful when there are a lot of external users: 

  • To group multiple external users (e.g. when selecting an external user).
  • To grant access rights to all external users of a group at a time.

Groups are combined in a flat list (where groups are independent from each other) or in hierarchy. IN an hierarchical list, external users of nested groups are included in higher-level (parent) groups.

There is a predefined group, All external users, that includes all external users. By default, this is a current group. 

When you assign roles directly, you can do it for each external user or for a group of external users.

Grouping external users

You can generate groups of external users, e.g. according to their relation to application objects:

  • Counterparties
    • Trusted partners
    • Main customers
    • Main suppliers
    • Survey respondents
  • Individuals

You can create external user groups that will include members of one type, e.g. respondents. It makes setting access rights

easier.
