External users groups catalog

Viewing and editing properties of a group of external users.

You can divide external users into groups by various criteria. An external user group can include users of the same kind, e.g. counterparts or individuals. You can create an external user group that will include different external users with similar properties, e.g. respondents. That simplifies the access rights setup procedure.

Adding external user group

  • Enter the Name of the group.
  • In the Group members field, you can define the members of the external user group. A group can include:
    • Any external users with no restrictions - select the Selected users value.
    • External users of a certain kind (e.g. counterparties) - select Selected users of one kind and specify the kind.
    • All external users of a certain kind (e.g. all counterparties) - select All users of one kind and one of the suggested kinds. In this case, the Members list can not be edited, since all the external users from this list are included in such group automatically.
  • When adding and deleting external users in the group, remember that they belong to parent groups.
  • To add a lot of external users , you can use the Select command. IN this case, you add several external users at once and add them to the group.
  • When you use the Add command, you can add only one external user.
  • You can subordinate the new group to one of the existing groups. IN the In group field, select a necessary group from the list.
  • On the Comment tab, you can write a detailed description of this group.

Access rights of external user group

According to the content, you can configure the applications as follows:

  • Click Access rights on the group navigation bar. Click Include in group, select an access group from the list.
  • If you assign roles directly, you can do it for external users and for an external user group as well. 

Viewing inactive external users

  • External users may be marked as inactive. By default, they are not shown in the lists. To view such external users, use the More - Show inactive external users command.
