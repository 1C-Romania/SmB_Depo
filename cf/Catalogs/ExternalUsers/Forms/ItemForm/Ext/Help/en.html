External user properties.

To enter or edit the external user data, you need administrator rights or the rights of a person responsible for the external user list.

Only the administrator can allow to log on to the application.

The person responsible for the external user list can maintain the External Users list to a limited extent, configure the external user before the application entry is allowed by the administrator. The responsible person can use the External access command, edit the External users and External user groups lists. The responsible user also edits the access rights for the external users.

In other cases a user can only view his personal data and change the Password (depending on the administrator settings), interface language, and contact information.

In a distributed infobase you configures the application logon for each infobase of the application.

Recommendations for logon password creation

A good password shall:

  • be illegible
  • have over eight characters
  • include the following:
    • upppercase latin letters
    • lowercase latin letters
    • numbers
    • symbols (highlight, brackets etc.)

The password shall not repeat the user name, consist of numbers only, contain legible words, or repeating character groups.

  • Good password examples:
    “nj7{jhjibq*Gfhjkm,
    F5”njnGhkmNj;t{HI.
  • Bad password examples:
    Ivanov,
    qwerty,
    12345678,
    123123123.

Adding an external user (if permitted)

If the user responsible for the list adds an external user and fills in the user name, the administrator will receive the recommendation to check the logon settings. When logon is allowed, the recommendation is closed, and the responsible user can not edit the logon settings, but disable the login permission. To view all such external users prepared by the responsible user, you can filter the list with Prepared field.

  • An external user is linked to a certain application object (eg., counterparty, employee, etc.) specified in the Object field. When using the External access button (recommended) the Object field is filled in automatically.
  • When adding an external user from the external user list using the Create command you should fill in the Object field manually. To do this, click Select, select the application list, and then select the required object from this list to which the external user is linked. The Object field will be displayed in various lists of the application as the external user's full name.
  • By default, the Invalid check box is cleared for all external users. Sometimes an external user does not use the application any more, and you can not deleted the user because this user referenced from other application objects. IN such case it is recommended to select the Invalid check box. Such external user can not be selected or seen in the external user list.
  • Enter general data of the external user on the Main tab:
    • Name (for logon) - the short name used to log on to the application
    • 1C:Company authentication -  the check box is selected by default and the password field is active
    • Password - enter your password to log on to the application. To protect your data, always fill in this field
    • Password confirmation - enter your password again to ensure its correctness.
    • User can not change the password - if necessary, prevent the external user from changing the password
    • Authentication with OpenID - select the check box to allow the external user to use a single OpenID-protocol account. Meanwhile, you can clear the Authentication of 1C:Company check box
  • On the Comment tab, enter additional information about the external user.

Adding an external user to groups

  • If user groups are enabled in the application, you may include your external user to one or more groups.
  • Click Groups on the navigation panel, and then select one or more groups using the check boxes.
  • To cancel your selection, click Exclude from all groups.

External user access rights

  • Access rights can also be configured by the user responsible for the external user list. However, some access rights (administrative rights) are not available for him. Depending on the application access rights are set up with one of the following options:
    • Click Access rights on the navigation panel to include your external user to one or more access groups. Select them from the list.
    • In the Allowed actions (roles) list you can assign one or more roles to your external user. All together they constitute his personal access settings. Roles may correspons with the user's various job duties (or activity kinds) or less important functions. Select relevant roles using check boxes.

Allowing application log on (only for the administrator)

  • For an external user to log in to the application the administrator selects the Application log In Is Allowed check box. Be sure the necessary properties are specified: name (login), password and other authentication parameters.

Managing external user settings

  • The application remembers the settings of each user when he works. To view, copy or delete external user settings (you have to delete such settings when incorrect settings prevent normal functioning of the application), click Settings on the navigation panel.

Application warning

  • If the properties of an external user differ from the properties of an infobase user, the application gives a warning showing such properties. You are invited to record the data to eliminate the differences.
