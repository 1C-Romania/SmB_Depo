For which purposes does the rule of the petty cash exchanges serve?

According to the rules of exchange the goods are loaded on a petty cashes operating in the Offline mode. For each petty cash there must be at least one rule. The same rule can be assigned to multiple petty cashes.

What is necessary to fill in in the card of the rule of the petty cash exchange?

In the card of an exchange rule it is necessary to specify the name of the rule in the field of "Peripheral Type" select "CR Offline", and it is obligatory to specify the warehouse. For the goods exported at the petty cash the application automatically assigns a code and in the rule it is possible to restrict the list of the goods by specifying "MAX code." IN the rule it is possible to set the selection of items, for example, it is possible to import only that items positions which are available on the specified warehouse (tick next to "only in stock").

How to view the list of goods taking part in the petty cash exchange?

The link "Show the list of items to edit" opens the list of goods involved in the exchange with CR ( in accordance with the specified filter) . IN the list it is possible to edit the automatically assigned item code ( according to which the product will be loaded to the petty cash). Using the command "Administration" - "deletion free codes at the end of the list " from the end of the list will be removed the lines with a blank items. When filling in the list it is performed a barcode and a price value checking. The price value is checked according to the type of prices which is specified in the card of the selected warehouse. If the for the selected items IS not set the prices or barcodes then the information about this is displayed in the corresponding columns of the list. If in the rule it IsSet to limit the list by the maximum code then the goods exceeding the maximum code are not going to participate in the exchange and will be highlighted in red. By the command "Update" the list of goods involved in the exchange with the CR will be recorded.

What is necessary to fill in in the card of the weights exchange?

In the rules og exchange card it is necessary to specify the name of a rule, in the field "Peripheral type" select the value "Labels printing Scales", obligatory to specify the warehouse. For the goods exported to the weights the application automatically assigns a code (PLU), and in a rule it is possible to limit the list of goods by specifying the "MAX code". Also it is necessary to specify the "Weight Product Prefix". The prefix IsSet as the second digit of the bar code (on the left side) of the weight product. The first digit of the barcode of the weight product always is equal to two. Thus, by the first two digits of the barcode the cashier can distinguish between weight and piece products. About the value which has to be specified in the "Weight Product Prefix" field   see in the documentation to the cashdesk and software Seted at the cashdesk. Also in exchange rule it is necessary to specify the unit of measurement  of the weight (select the unit of measurement  from the filled field "Unit of measurement type" by the value "Weight"). IN the weight loading will take part only products with the same unit of measurement.

How to view the list of products taking part in the weight exchange?

Using the link "Show the products list for editing" it is possible to open the list of goods taking part in weights exchange(according to the filter Seted). IN the list it is possible to automatically edit the assigned code for the product (according to with the product will be loaded to the scales). By the command "Administering" – "deletion the free codes in the end of the list" from the end of the list will be deleted the lines with the empty items. When filling in the list the application is checking the availability of the product price. The price value is checked according to the type of prices which is listed in the card of the selected warehouse. Barcode for the weight products will be generated in the process of exporting and automatically, it is not required to generate it in advance. If the rule is configured to set the limit of the list by the maximum code then the products exceeding the maximum code will not participate in the exchange and will be highlighted in the list in red color. Using the command "Update" the list of products involved in the exchange with weights will be recorded.
