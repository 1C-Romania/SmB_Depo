Designed to maintain users working with the application.

Open the user list from the Administration - Users and right settings or Access management section using the Users command.

The user list can be maintained by the administrator and the user responsible for the external user list. 

Login may be accessed only by the administrator.

The responsible user (permitted to add new users) can partly maintain the list and set up users before access is granted to the administrator. The responsible user can edit the Users and User group lists.

All users can view the information about themselves and change several properties: a password, a application language, and contact information. 

The user added by the list-responsible user gets the Prepared status. The application administrator may select such users by this criteria, check for data accuracy, and allow logging in.

After the administrator allows login, the list-responsible user can not edit the access settings except for disabling login permission.

In a distributed infobase, login is configured individually for each application infobase.

If there are a lot of users, the list may be grouped. For example, you can use user groups to add a company departments structure.

You can use the left-click context menu to work with the users list.

Adding user

  •  Click Create, fill in the required fields.

Viewing invalid users

  • You can mark users as invalid. To view such users, select the Show invalid users check box at the bottom of the list.

Viewing subordinate group users

  • To view all the users included in subordinate groups according to the hierarchy, select the Show subordinate users check box at the list bottom. By default, the check box is selected.

Viewing infobase user list

  • To ensure that the data is entered accurately and to correct incorrect input data, use the More - Infobase users command to control entries and synchronize the user list with the list of infobase users.

Grouping users

  • If you are allowed to use user groups, the list consists of two parts: in the left, a necessary quantity of groups; in the right, a list of users of the selected group. By default, the current group is All users, i.e. all users are visible.
  • One user may be included in different groups. You can add a user to the group as follows:
    • When adding a user to the list, you can select a group, and then click Create.
    • To add more users to the group, you can open the group window and click Pick. The pickup process uses the Users list, consisting of three main parts.
    • Select a user from the list, use the More - Assign groups command and select the check boxes to add them to one or more groups. You can also find this command in the context menu.
    • Drag one or more users to the necessary group. Confirm your selection by clicking Yes. To use this option, clear the Show users of subordinate groups check box.

Adding user group

  • To add a group, click Create group.
  • You can use the Create command of the context menu in the left part of the list.

Adding user group to another group

  • Select a group in the left list part, use the Move to group command of the context menu.
  • Select the group to move.

Selecting user group members

  • If you select group members from the user list, it consists of three parts.  IN the upper left part of the list, you can see groups; in the lower part - users of the selected group. IN the right, there are the selected users.
  • To simplify the upper left part of the list, the existing user groups are shown. Select the necessary group (or the All users group) to view the users of this group in the lower left part of the list.
  • Select one or more users in the lower left part, use the arrow to move the Selected users to the right part. You can also move a user from the lower left part to the right part with a double-click.
  • To cancel your choice, remove one or more unnecessary users from the left part through similar steps.
  • Use the << double arrow to delete all users from Selected.
  • Selected users are counted automatically.
  • When the user group user selection is complete, click Complete and close.

Viewing user data report

  • Use this report to analyze the login settings (IB user properties) together with other user properties in the catalog. There are three report variants: about users, external users and all users.
  • Click More - User data and generate the report.
