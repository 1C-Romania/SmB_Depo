Designed to manage file storage volumes.

It is used if the file storage mode is In volumes on drive. To set it, in the Administration - File settings section, select the Store files in volumes on drive checkbox. Then the Administration - File settings section enables the File storage volumes command which opens the list.

The list contains:

  • Name- a name of the file storage volume.
  • Order - an integer. The volumes are filled in ascending order until the volume is full.
  • Full path under Microsoft Windows - a path to the volume directory of the server under Microsoft Windows in the UNC format: \\Computer name\Folder name\File name\.
  • Comment - detailed information about the volume.
  • Limit, MB - you can specify the maximum size of all the files in the given volume (volume quota). If the quota is null, files are stored in the volume without checking it. IN case of a more than null value of the Maximum size and the total file size over the predefined quota, the successive volume will be used. If no available volume has enough space to store the file, the information about it will be recorded in the registration log, and an error occurs.
  • Full path under Linux - a path to the directory for the server under Linux.

Adding file storage volume

  • Click Create and fill out necessary fields.
  • Specify one or more directories on the hard drive of the server on which the application operates or of other network servers that will store all files.

Working with file storage volumes

  • When creating a new volume, access to the directory specified in the Full path field is automatically checked. If an error occurs, the relevant message appears. 
  • The volumes are successively filled out in ascending order. You can use this feature to temporarily disable a volume by increasing its filling order.
  • You can not change paths to the volume you have already created.
  • You can not delete the created volume while some file versions are linked to it.
  • Volumes access errors are stored in the registration log and can be seen by the application administrator.
  • Every volume stores files in subdirectories by date, so that a separate subdirectory is created for every day. All files in the volumes are stored with the original name and extension; the version number is put between them, e.g. "Order.1.txt".
  • When changing the file storage mode (e.g. to In volumes on drive), all the existing files are not moved to the volumes - only new files will be stored in the new way. To move the existing files from the infobase to the volumes, use the Move files to volumes command on the action panel of the Organizer section.
  • If data is synchronized between the source application and recipient application while the source application stores files in volumes on drives, you must create an archive file with volumes of stored files when you create an initial image.
  • While getting the initial image in the recipient application, specify the location of the archive file with volumes of stored files. To get files in volumes from the archive, run the Place files of initial image command on the action panel of the Organizer section.

Note
To use volumes on the network drives, launch the 1C:Enterprise 8 server as a domain1\user1 domain user with access to the volume specified in the UNC path. This user must belong to the Users group of the local computer, have the "Run as service" and "Run as batch job" rights and the rights to the directory where the 1C:Enterprise 8 server stores the service files: registration log, temporary files directory, etc.
