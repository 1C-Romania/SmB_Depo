A file card contains information about a file and its current version. It also presents buttons for basic actions with the file. You can find additional commands for operations with the file in the More menu.

To open a file card, press F2 or click More - Open card in the file list. You can also open a card in the list of attached files of a application object, full-text search or reports.

File data

  • Name- a full name of the file.
  • Description - detailed description of the file.
  • Created - time and date when the file was added to the base.
  • Author - a user who added the file.
  • Edited by - a user who locked the file for editing.
  • since - date and time when the editing started.
  • Folder - a folder where the file is saved. Use the button to select another folder.
  • Code - a file code.

Current version

  • Number - a file version number.
  • Author - a user who added the file version.
  • Created - date when the file version was added to the base.
  • Changed - date when the file data was changed (local time).
  • Size (byte) - a size of the file version data.
  • Extension- a file format; the sign near it also indicates the format.
  • Encoding  - the file encoding.

Storing versions

  • Select the Store versions check box to create a new file version after you finish editing. When the check box is cleared, the stored file is updated. It is recommended to store versions, since if a file has been damaged during editing, it can be restored from its version. By default, the check box is selected.

When you use digital signature and encryption, you work with the following tabs:

Digital signatures

The tab contains data about the digital signature of the file:

  • Signature author - the user who signed the file.
  • Signature date - the file signature date.
  • Comment - additional information about the signature.
  • Status - signature validity.

Use the relevant commands to:

  • Sign the file
  • Open signature - open the window with the main properties of the digital signature of the selected file: issued by, issued to, certificate purpose and validity period.
  • Check if the file has been edited with one or all the digital signature certificates. To do it, select one or more signatures. If a digital signature does not correspond with any digital signature certificate, such a digital signature, when checked, will be highlighted with red and have the comment Incorrect in the Status column.
  • Check all - check all signatures
  • Save a digital signature in a file on the drive
  • Delete a digital signature of the file

Decryption allowed

  • The tab contains file encryption data:
  • The list displays certificates used to encrypt the file. Double-click or use the relevant command to Open certificate of the digital signature and view its main properties: issued by, issued to, certificate purpose and validity period.
