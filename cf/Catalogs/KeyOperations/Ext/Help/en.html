It is intended to save the key operations parameters.

Key operations parameters are necessary to estimate application performance. The application includes embedded tools for data collection and analysis of work performance according to APDEX - an international standard for estimation of the information systems performance.

You can open the list by the Key operations command of the Service subsystems section.

It is necessary to fill key operations parameters when application starts to estimate application performance (key operations parameters are not filled after application installation). There are predetermined key operations in the application. You can add operations in the list if necessary. Application predefined operations are listed, for example, document posting, report generation etc. The list includes:

  • Name - operation full name;
  • Priority - an integer number, at that, the priority must not be repeated;
  • Target time - time spent for operation in seconds;
  • MIN allowable level - numerical estimates are taken for comparison according to apdex , for example, Satisfactory (apdex > = 0.75).

Key operations selected parameters will be applied during the performance measurements.

Filling the key operations data

  • Allocate available key operation in the list, click , and then fill necessary fields.

Entering the new key operation

  • Click Create, and then fill necessary fields.
