Registration of additional reports and processors.

Some data in the additional record card or the processor card are provided by default. The developer specifies it in the external processor file or the report file. You can not change this data (the commands list and additional information). Each additional processor can represent one or more commands placed in the application objects windows or in the application sections.

There are two types of additional reports and processors:

  • Assigned - designed for one or more specific documents or lists. The list of available assignations is predefined by the developer and included in the additional report or processor. When registering, you can select assignations from the list.
  • Global or common - contain commands only from the command interface sections. The command does not depend on specific application objects. The list of the sections available for commands placement is predefined by the developer and included in the additional report/processor file. It can not be edited.

The additional reports/processors may be numerous, according to the assignation and settings. The required commands are placed on a special tab. The required Additional information tab includes information about the additional report/processor. Other tabs may have different content and names. The additional tabs may also contain special commands. When registering an additional report/processor, specify the following main parameters on the tab:

  • Name- stated by the developer and entered automatically; it can be changed when necessary.
  • Operation mode - the mode of connection of additional reports/processors; it can be either Secure or Insecure.

  • Use the Publication checkbox to make the processor/report availabe/unavailable - adjust the usage of the additional processors/reports. You can make a processor/report available to all, or for the administrator only, or disabled. The attribute can have one of the following values:

    • In use - the processor can be used. The processor commands are available to all the users with the "Using additional reports/processors" role.
    • Debugging mode - the processor is available only from the "Adding/editing additional processors" or "Full rights" roles (highlighted with red in the list). Other users do not see these commands. This mode is useful if your additional report/processor is to be checked.
    • Disabled - the processor is unavailable, inactive in the list. This option is useful when you suspend using a processor, but do not want to delete it. All the settings are saved and when you go to another mode, it works like before.
  • Comment - enter a detailed description.
  • Responsible user - select the user that will monitor and update the additional report/processor. By default, it is "Administrator".

Additional information

  • You can see the following additional data entered by the developer:
    • Kind - there are several kinds of additional reports/processors:
      • Additional report
      • Report
      • Completing object
      • Print form
      • Creating linked objects
    • File name
    • Version
    • Object name
    • Description - detailed information about the report/processor.
  • Group - add the additional processor/report in one of the group. Select a necessary group from the list. Using groups, you can easily find required additional reports/processors in a list.

Exporting from file

  • Use Export from file command to update the processor in the internal storage. Kind of the new processor must be the same as the current processor kind. Specify the path to the file, and then click Open.

Importing to file

  • Use the Import to file command to import the additional report/processor from the storage to a file on the computer disk. Specify the folder on the computer, and then click Save. The file is named automatically, but you can rename it.

Operation mode

  • Normally, most processors are executed in the safe mode. It guarantees that they do not contain any insecure application code being executed on the server.
  • Still, some processors may require execution in unsafe mode. Export such processors only from reliable sources (suppliers) and check them before export.
  • For more information on the safe mode, see 1C:Enterprise documentation.

More about registration of assigned additional processors/reports

  • For additional processors/reports intended for the application lists/documents, you can select objects with access of the commands of the report/processor.
    • Purpose -  to select application objects, click the link seen as Undefined (if no objects are selected) or a list of the selected objects. Select the checkboxes to select the necessary objects.
    • Show - the reports/processors commands are available from:
      • List of selected objects if the In lists check box is selected.
      • List of the object if the In forms of catalog/document items check box is selected. You can select both check boxes at once.
  • On the Commands tab, the commands list is by default completed from the developer's list. These commands are available in the assignation list/document when adding the additional report/processor.

More about registration of global (common) additional reports/processors

  • For the processors/reports invoked from the command interface (not attached to configuration objects), specify the interface sections where the processors commands will be available through the Additional reports and Additional processors commands.
    • Placement- to select sections, click the link seen as Undefined (if no sections are selected) or a list of the selected sections. Select the check boxes to select the necessary sections. Click OK to confirm.
  • The list of commands and their settings can be located on a separate Commands tab. The tab shows:
    • Name - a full name of the command.
    • In the Quick access column, click or press F4 and select users with default access to the command. By default, the value is "No". Add the user to the list. Click OK to confirm your choice and return to the cofiguration procedure.
    • In the Schedule column, for each command of global reports/processors executed in the server context, click and define the schedule for the scheduled job of this command. You can not use a schedule for the commands started with Opening form.
