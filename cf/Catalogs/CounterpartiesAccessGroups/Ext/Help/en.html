Counterparties access groups

The catalog is intended to describe access to Counterparties catalog items in accordance with kinds of access provided for by the Counterparties group. The catalog is used in cases when it is necessary to restrict access of users to certain specific counterparties. The information as to which access group this or that counterparty belongs to is given in the Counterparties catalog. It is possible to assign counterparties to a certain access group with the help of the changing details for groups service mechanism which can be run for selected counterparties from the list form of the Counterparties catalog.

Example. Sales managers work with various counterparties. Some managers work with wholesale buyers, other work with retail buyers. Managers shall have access only to information related to the counterparties which they work with and they shall not have access to information on counterparties of other sale managers. Data on all counterparty groups which sale managers work with (Wholesale buyers and Retail buyers) is entered to the Counterparties Access Groups catalog. As a result, the corresponding access group - Wholesale Buyers or Retail Buyers - is assigned to every user as an authorized counterparty access group.

Thus, sale managers will have limited lists of counterparties available to them:

  • managers working with wholesale buyers will see only the information related to counterparties for which the Wholesale buyers counterparties access group is defined
  • managers working with retail buyers will see only the information related to counterparties for which the Retail buyers counterparty access group is defined


Important. Counterparties access group restrictions can be set only if in the Settings/Settings of Users and Rights section the Restrict access at entries level and Counterparty access groups options are selected.
