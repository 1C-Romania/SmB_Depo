How to set up the signing route of an electronic document?

In case it is assumed, that EDS will be used on electronic documents exchange, you need to configure the settings in the EDS Certificate form as follows:

  • Signing certificate will work only for electronic documents executed on behalf of the Company.
  • The Kinds of documents table specifies a list of document kinds for the certificate to be used.
  • The List of performers table defines a list of performers for the certificate who can send electronic documents to the authorised person - a user, that has a right to sign documents with this certificate. If the Check the executors structure check box is not selected, then any system user can send a document to authorised person for signing with this certificate.
  • If it is nesessary to provide access to the certificate for one user only, then you need to select the Limit access to certificate check box and select a user.

IMPORTANT. After all settings for the certificate are configured, it is recommended to verify them using the Certificate settings test command. This command will test the possibility to use this signing certificate for cryptography operations on this computer in current infobase on behalf of the current operating system user.

How a user can enable the signing of electronic documents option without password request?

To sign electronic documents without certificate access password request, you need to select the Remember password for certificate check box, and then enter access password for private part of the key.

IMPORTANT. It is nessesary to limit physical access for unauthorized persons to the private key part by selecting the Remember password for certificate check box. Otherwise there is a chance for unauthorized signing of an electronic document.
