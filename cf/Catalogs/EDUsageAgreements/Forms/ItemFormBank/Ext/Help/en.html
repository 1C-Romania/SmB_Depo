Correct identification of participants in direct exchange setting for banks

For direct exchange setting for banks, do following:

  • Select an Organization
  • Define a Bank for interaction

According to the selected bank, you see a set of required parameters.

If you have selected a bank which parameters are unknown, you can select a bank service connection variant.

There are three connection variants:

- Direct connection to bank service (asynchronously). Data request and data receipt are divided into separate processes.

- Direct connection to bank service (synchronously). After a data request, the system always waits for a response from the bank server. IN this case, there are two possible authentication variants: With certificate and With login and password.

For this variant, specify the Bank resources for data exchange on the Data exchange settings tab: bank web server address, data sending/receipt resources addresses. The bank provides values of these parameters when making a contract with the customer.

- With an additional data processor - exchange with the bank uses an additional data processor.

To use this variant, enable additional reports and data processors. Then export an external data processor from a file and specify it in the agreement. The file with a data processor is provided by the bank.

Important. To exchange electronic documents with a specific bank, enter only one agreement for each organization. This agreement will use the settings for all the bank accounts of the organization.

Direct exchange in asynchronous mode

This variant generates electronic documents and sends them to the bank without a signature. After sending, payment documents are available in the external client bank. To have a payment executed, confirm it in the client bank. To log on to the bank resource, use your login and password: use the EDF settings form to:

  • Select the bank application: Direct connection to bank server (asynchronously).
  • Enter the organization ID
  • Specify the bank server address

Direct exchange using digital signature

To use electronic document signature for direct exchange with banks:

  • In common settings, enable digital signature
  • Define general cryptography settings
  • Select the connection variant: Direct connection to bank server (synchronously)
  • Specify the Bank server address, resources for sending and receipt of electronic documents
  • Specify the authentication variant - with signature certificate
  • In the EDF settings form, in the table segment of the Digital signature key certificates, specify items of the Digital signature certificates list; set up the sequence of the organization's "first" and "second" signatures in electronic documents.

Important. The sequence of the Digital signature certificates items in the list defines the owner of the "first" and "second" signatures. If the organization has only the "first" signature in the payment order, specify only one digital signature certificate in the agreement settings.

Direct exchange with additional data processor

To use this variant, enable additional reports and data processors. Then export an external data processor from a file and specify it in the agreement. The file with a data processor is provided by the bank.

In this case, all the cryptography, authentication and data transmission functions are performed with an additional data processor. To use this variant:

  • In the application settings, enable additional reports and data processors
  • Add an additional data processor to the Additional reports and data processors catalog
  • Select the imported data processor in the EDF settings
  • In the EDF settings form, click Export certificate from storage to add necessary signature certificates
