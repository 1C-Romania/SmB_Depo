Generally, access group profiles are configured during implementation and configuration of the applied solution. They are designed to create complete sets of allowed actions (roles) with specified access rights and access rights limitations (access kinds) to be used in access groups.

Sometimes, predefined access group profiles are not enough for necessary user access rights settings.  In such cases, the administrator can add or edit access group profiles.

Use profiles to:

  • Create a common list of roles for a certain user group. For example, managers may have an access group profile called Manager including the following roles:
    • Adding and editing customer orders
    • Reading reference data
    • ...
  • Define the access kinds you use to access certain list items, documents, etc. available for the roles from the above list. For example, the Manager profile may allow (or forbid) access to specific customer orders, reference materials and other application data in terms of organizations, partners, operation types, and other available access kinds. Specific values for organizations, partners, operation types, and other access kinds defined in the profile are configured in the relevant access groups.

Access kinds and limitations

  • According to their character, roles may or may not support access limitation within certain access kinds. For example, the Adding and editing customer orders role may limit adding and editing customer orders in terms of organizations and partners, while the Starting web client role allows starting a web client regardless of anything.
  • Therefore, when you set up access kind content in the profile, remember that a profile must have a comprehensive list of access kinds for all the roles in it. 
  • If a certain access kind has not been specified in the profile, limitations will not apply to it (as if it was not used in the role access limitation logic).

Application administration profile

  • The application contains the predefined Administrator profile having only one role, Full rights. This role provides full access to application data with no limitations. It is not recommended to add the Full rights role to any other profiles, since it automatically disables any access limitations (a profile with this role "turns into" Administrator).
  • The Administrator profile is used in the predefined access group: Administrators. It is not recommended to create other access groups with this profile.
