Designed to store reports variants and configure their arrangement in the command interface.

Report variant data

  • Fil in the Report variant name that will be displayed in the application lists and in the reports panel.
  • Write a description with detailed information about the report variant; it will be used in pop-ups in the reports panel and in the reports variant lists.
  • Select the Available check box to define who will see this report variant:
    • Only for author - only the author can see the report variant; click Select to select an author from the user list. If necessary, you can select another author for the report, e.g. when this report is referred to another responsible user.
    • For all users - available to all the users of this report.
      • Visible in the reports panel by default - select the check box to always show the report to all the users of the reports panel.

Locating in report panels

Administrators and authors of the reports variants can change their location.

  • Select one or more check boxes in Section, group column in the relevant sections or application groups. When placed in a section, the report variant will be shown at the top of the section reports panel, without placing in groups.
  • In the Importance column, select the method to mark the report in the group:
    • Important - a report variant will be bold.
    • See also - the report variant will be displayed in a separate group at the bottom of the reports panel.

Generating report

  • Click Open, specify the necessary report parameters, and then click Generate.

Resetting settings

  • You may need to reset the settings if they are erroneous and impede your work with the report. Use the following commands:
  • More - Reset placement settings - reset the placement settings for this report variant.
  • More - Reset user settings - reset the user settings in the reports panels for this report variant.
