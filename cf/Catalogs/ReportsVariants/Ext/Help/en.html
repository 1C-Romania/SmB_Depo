Designed to manage the reports variants, configure the placement in reports panels of the application sections.

You can save and use multiple variants of the same report. Predetermined reports variants are available. You can create your own reports variants by editing the settings of the existing ones: generation and appearance settings (field content, sorting, grouping, conditional appearance).

You can also create common reports variants. Common reports variants are available for all the users of this report.

You can navigate from the application sections to the reports panels available in these sections. If there are a lot of reports, they are distributed among the section groups. To make your reports panels more convenient, configure them, e.g. hide the rarely used reports variants and setup quick access to the most popular panels.

The reports variants administrators have extra options for configuring the reports panel appearance: they can mark a report variant as important and put it into the See also group.

Open the list with the All reports command in the reports panel of the application section. The sections and groups in this list depend on the available permissions and roles. Only the application administrator has full access to all the reports variants.

The application administrator can open the list by clicking the Reports variants command in Administration - Print forms, reports and processors. Some commands or actions available for the administrator can be unavailable in the All reports list. These features are mentioned in the list capacities description.

The list consists of two parts:

  • In the left part, there is a tree of command interface sections and groups having reports panels.
    • As you navigate between the tree nodes, the list is automatically filtered by a selected section or group.
    • Use the Including subordinate check box to show all the reports of a section or a group, including the reports variants in the nested groups. 
  • The reports list is in the right.

The list displays:

  • Name - a full report variant name that will be shown in the lists.
  • Author - a full name of the author of this report variant.
  • Description - detailed information about the report variant.

To add columns to the list, click More - Edit form (the settings are saved):

  • Image Available to all the users:
    • Shown - the report variant is available to all users.
    • Hidden - the report variant is available only to the author.
  • Technical information:
  • Report name - a report registration name.
  • Report - a full report name (representation).
  • Report type - there are three report types:
    • Internal
    • Additional
    • External - variants of external reports can not be located in the reports panel.
  • Variant key - a name (unique ID) of the predetermined user report variant. 
  • Parameters and filter names - a list of report parameters and filters.
  • Field names - a list of the report field names.

    Search

    If the reports variants list is long, use search to find a necessary report variant quickly.

    • In the Search field, enter a necessary value, and then click Search. The selected section (group) will be searched.

      • You may search reports by multiple words, a word, or a part of word. Search is carried out:

        • In report names
        • In report variant descriptions
        • In representations of the report variant fields
        • In representations of parameters and filters of the report variant
        • In names of user settings of the report variant
        • In names of the sections and groups of the application and reports panel.

    Generating report

    • Select a report variant, click Open, customize necessary settings, and then click Generate.
    • You can also double-click a necessary report variant in the All reports list.

    Customizing report variant

    • Select a report variant, click Edit, and then edit necessary fields.
    • In the Reports variants list designed only for the administrator, double-click a necessary report variant.

    Deleting report variant

    • To mark a user report variant for deletion, in the More menu, click Mark for deletion or press Del.
    • Repeat this command to clear Mark for deletion. 
    • Predetermined reports variants do not have this option.

    Placing report variants in application sections 

    The report variant may be placed by the author or the administrator.

    • You can drag reports variants from the current section/group. Hold Ctrl while dragging the report variant to add it to a new group and keep it in the current group. The application prompts for confirmation of the report variant placement. Click Yes to confirm.

    • Use the More - Place in section command to configure the placement of multiple reports variants: specify the command interface sections and groups where they will be placed.

    • You can use extra features for configuring the reports variants appearance: mark a report variant as Important or move it to See also.

    •  In the administrator's Reports variants list, you can put the Place in sections command to the list command panel for better convenience.

    Resetting settings

    • Click More - Reset placement settings to reset the initial settings of the selected reports variants: placement in the application sections, default access and visible. The command can be run only for predetermined reports variants.
    • Click More - Reset user settings to reset the user settings of the selected reports variants: visible and quick access.
