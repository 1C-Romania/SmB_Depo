Designed to maintain user groups. Groups are useful when there are a lot of users:

  • To group a lot of users (e.g. when selecting a user).
  • To configure the common access settings for all the group users simultaneously (e.g. you include the accountants in the Accounting department group; then you set access rights to various lists and documents for this group).
  • To configure common settings in other cases, e.g. in the roles of the business process task performers, etc.

Groups are created in a flat list (they do not depend on each other) or in an hierarchical list (groups can be on different levels). IN an hierarchical list, the users of nested groups belong to higher-level (parent) groups.

The list includes the predefined All users group containing all the application users. By default, this is the current group. When roles are assigned directly, you can assign them to each user as well to the entire user group.

Grouping users

You can create user groups that will contain users of one type. It is useful when setting access rights. For example, groups can be related to the company structure, job positions and project teams, such as:

  • Enterprise
    • Management
      • Department 1
      • Department 2
    • Managers
    • Engineering and technical services
      • Department 1
      • Department 2
        • Group 1
        • Group 2
      • Engineering and technical services managers
    • Production
      • Workshop 1
        • Land 1
        • Land 2
      • Workshop 2
        • Land 1
        • Land 2
        • Land 3
      • Production managers
    • Enterprise managers
    • Group of Joint inventory
