It is intended to Save and view metadata objects identifiers. The data of the list is not editable and is intended to ensure application operation.

The list is filled in automatically at the first start (and each update) of the configuration according to configuration metadata, it is also automatically updated each time when configuration version is updated (renamed, added and deleted metadata objects are considered). All application objects grouped in types are listed in the form of a tree. The Allowed action (role) object field is displayed by default. By running the More - Change form command it is possible to add the Full description and Identifier fields into the list.
