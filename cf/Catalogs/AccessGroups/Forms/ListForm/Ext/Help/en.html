Use the access groups list to maintain access groups and view current access settings. To open the access groups list, click Access groups in Administration - Users and rights settings. Click Access groups in the access group profile navigation panel to navigate to the list of access groups created from the access groups profile.

An access group defines actions with application data allowed to the group members. Usually, access groups correspond with the job duties (occupations) of the application users. A user can be a member of different access groups which compose their personal access rights settings.

Use access groups to create and administer common access settings for a certain group of users with similar duties, e.g. "Central office accountant" or "Sales manager at X office".

Access management requires administrator rights. The user responsible for an access group can add or delete members of their access group.

The list shows:

  • Name - the access groups are listed alphabetically.
  • Profile - the list can show the access group profile.
  • Responsible user - this column shows the user responsible for a given access group.

Adding access group

  • Click Create and fill in the required fields.

Adding access groups folder

  • You can create folders (groups) to find a necessary access group more easily and understand the access constraints for the whole group, since there may be many access groups with alike rights constraints.
  • Click Create group and enter the Name of the folder.

Adding and deleting access group members

  • Select a necessary access group and click More - Change.
