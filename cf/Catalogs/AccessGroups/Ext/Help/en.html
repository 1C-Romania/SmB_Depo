Access groups are designed to set access rights and restrictions for specific users and user groups.

You can set access to various functions, lists and documents. Predefined sets of roles are available. You can also configure access manually. Access rights are managed by the administrator. The user responsible for an access group can add and delete its members.

The set of rights and restriction options depend on the access group profile.

For example, a "Business software sales managers" access group containing users Smith and Johnson may be linked to a "Sales manager" profile that includes a Products and services kinds access kind restriction. If you select "Prohibited" for all values of this access kind and add the "Business software" products and services kind to the list - Smith and Johnson will have access to data and operations only if they are related to the business software.

The group names should reflect the profile idea and the group restrictions.

A group may contain individual users (external users) and user groups (external user groups). If a lot of users are members of several different groups simultaneously, it is recommended to divide users into groups. When you include such groups in your access groups, you can avoid routine review of the members of several access groups.

Set of user roles

  • Sets of user roles are determined automatically on the basis of the direct and indirect (via user groups) membership in access groups. It is updated every time when members of the access group or the user group (external user group) are changed. 
  • It is not recommended to change the set of roles in some other way (e.g. with Designer), since the set or roles will be restored with the next regular record of the access group.

Merging access rights

  • The user can be simultaneously included in multiple access groups.
  • In this case, all their access rights are summed up (joined with "or" rule) from the access rights of each group. For example, the "Procurement managers" group allows the users to view the documents: "Goods  arrival" for X Company. The "Sales managers" allows the user to add "Buyer order" documents for Y Company.
  • Instead of the allowed values lists stated in the groups, it is the access rights which are merged, i.e. in this example, the user will not get access to view the goods arrivals documents simultaneously with access to add "Buyers orders" documents for both organizations.

Administering access group

  • Administer the application with the special access group: Administrators with the Administrator profile. When you add a user to this group, the Full rights role is automatically assigned to them. If the application contains other users with this role who do not belong to the Administrator group, such users will not have the Full rights role.

Warning

  • Do not use the configuration mode of 1C:Enterprise to set access rights.
