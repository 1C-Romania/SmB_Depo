Cost allocation document

The document is used for distribution of expenses on items manufactured by production department.

Document attributes:

  • Company - company, for which the document is Creatingd. It is selected from the Companies list. The attribute is displayed on the form if Accounting by Multiple Companies is enabled in accounting options.
  • Structural unit - production department to which expenses are applied. It is selected from the Structural unit list.
  • Cell - warehouse cell of the department from where items will be WrittenOff.
  • From - period start date.
  • To - period end date which is equal to document date.

  • Finished Goods - list of goods for which the expenses have to be distributed. Document table can be filled automatically (command bar button "Fill by tools"). Table fields:
    • ProductsAndServices - selected from the ProductsAndServices list.
    • Characteristic - if necessary, characteristic of the item may be selected in this field. It is selected from the Products and services characteristics list. The attribute is displayed on the form if characteristics are enabled for the items in accounting options.
    • Batch - if necessary, batch of the item may be selected in this field. It is selected from the ProductsAndServicesBatches list. The attribute is displayed on the form if batches are enabled for the items in accounting options.
    • UOM - unit of measure. It is selected from the UOM Classifier list or from the Units of Measure list.
    • Quantity - COUNT of items.
    • Specification - specification, containing information about the production technology. It is selected from the Specifications list. The attribute is optional.
    • Customer Order - Customer Order document. The attribute is optional.
  • Inventory - list of stock items that should be distributed onto finished goods and jobs. The table can be filled using pick method (button Pick on the command bar); using the Fill button (by standards - the table is filled by specifications specified in the table Finished Goods; By Balance - the table will be filled by stock balance of the department selected in document header); manual data input.
    • ProductsAndServices - selected from the ProductsAndServices list.
      Characteristic - if necessary, characteristic of the item may be selected in this field. It is selected from the Products and services characteristics list. The attribute is displayed on the form if characteristics are enabled for the items in accounting options.
    • Batch - if necessary, batch of the item may be selected in this field. It is selected from the Products and services batches list. The attribute is displayed on the form if batches are enabled for the items in accounting options.
    • UOM - unit of measure. It is selected from the UOM Classifier list or from the Units of Measure list.
    • Quantity - COUNT of items.
    • Reserve - COUNT of items in reserve.
    • Specification - specification, containing information about the production technology. It is selected from the Specifications list. The attribute is optional.
    • Customer Order - Customer Order document. The attribute is optional.
  • Inventory Distribution - list of stock items distributed onto finished goods and jobs. The table can be filled using the Distribute button (by standards - the table is filled by specifications specified in the table Finished Goods; By Quantity - COUNT of items from the table Inventory will be distributed onto finished goods proportionally to COUNT of finished goods in the table Finished Goods) of command bar of the table Inventory. Data can also be entered manually.
    • Quantity - COUNT of items being distributed onto finished goods.
    • Reserve - COUNT of items in reserve being distributed onto finished goods.
    • ProductsAndServices - finished good item onto which inventory should be distributed.
    • Characteristic - if necessary, characteristic of the item may be selected in this field. It is selected from the Products and services characteristics list. The attribute is displayed on the form if characteristics are enabled for the items in accounting options.
    • Batch - if necessary, batch of the item may be selected in this field. It is selected from the Products and services batches list. The attribute is displayed on the form if batches are enabled for the items in accounting options.
    • Specification - specification, containing information about the production technology. It is selected from the Specifications list. The attribute is optional.
    • Customer order - Customer Order document. The attribute is optional.
  • Costs - list of non-material costs that have to be distributed. The table can be filled using the Costs Costs fill by balance command bar button or manually.
    • Expense GL Account - GL Account where costs will be accumulated.
    • Customer Order - Customer Order document. The attribute is optional.
    • Amount - cost Amount in currency of accounting.
  • Cost allocation - list of non-material costs distributed onto finished goods and jobs. The table can be filled using the Fill by Quantity command bar button (cost Amount from the table Costs will be distributed onto finished goods proportionally to COUNT of finished goods in the table Finished Goods) of the table Costs. Data can also be entered manually.
    • Amount - cost Amount being distributed onto finished goods.
    • ProductsAndServices - finished good item onto which inventory should be distributed.
    • Characteristic - if necessary, characteristic of the item may be selected in this field. It is selected from the Products and services characteristics list. The attribute is displayed on the form if characteristics are enabled for the items in accounting options.
    • Batch - if necessary, batch of the item may be selected in this field. It is selected from the Products and services batches list. The attribute is displayed on the form if batches are enabled for the items in accounting options.
    • Specification - specification, containing information about the production technology. It is selected from the Specifications list. The attribute is optional.
    • Customer order - Customer Order document. The attribute is optional.
  • Note - any additional information.
