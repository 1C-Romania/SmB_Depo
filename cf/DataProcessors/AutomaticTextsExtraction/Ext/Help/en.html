It is intended to manage automatic text extraction.

If one or several working processes of the server work under Linux and a part or all clients are connected using web-client, then files with the text not extracted can be added in the database at that these files can not be found using full-text search based on the content. IN this case you should run thin or thick client, and run the command as follows: Full-text search management - Run texts extraction in the Administration section - Support and service - Scheduled operations on one of the client computers working under Microsoft Windows. You can also run Automatic text extraction command of the Organizer section action panel.

Setting parameters

  • Set Execution time interval - execution time interval is equal to 60 seconds by default.
  • Set Number of files in one portion - Default value is 100.
  • Number of files with unextracted text - the application counts automatically.

Text extraction

Texts can be extracted as follows:

  • by portions
    • Click Start to start texts extraction by specified portions with process interruption option
    • Click Stop to interrupt the process
  • from all files at once
    • Click Extract all
