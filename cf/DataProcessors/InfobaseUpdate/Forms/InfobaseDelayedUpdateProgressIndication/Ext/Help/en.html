It is intended for viewing the progress and delayed database update results.

When updating the database the processing of some part of data can be delayed and processed in a background mode during application operation. IN this case, the documents that are not yet processed will be unavailable. At the attempt to open such a document, the application displays a message informing that the document processing is temporarily locked, as the application update was not completed. Some reports can also be unavailable. At the attempt to generate a report, the application displays a message informing that the report can contain incorrect data as the application update was not completed.

It opens automatically when application starts after the database is updated along with Application revisions description (What's new in configuration) window.

Information on delayed application update results can be found in the Administration - Support and maintenance section by running the Update results and Information processing command; or by the Additional procedures in progress link in the What's new in configuration window. After running the command the link changes its description to the Additional data processing procedures are being performed.

Information on database updating

  • Update version, update date and time, updating process time are specified.

Viewing the event log

  • To view the database update events in the event log, click the Technical information on update results link.

Data processing activities

  • If data processing activities have not been launched yet, the link displays the Data processing activities list. Click the link to view additional procedures list.
  • Click Run to start data processing.
  • If the processing starts, the link displays the number of procedures completed.
  • When the data processing activities are completed, completion date and time are shown. The link displays All update procedures are successfully completed (the number of performed procedures is specified in parentheses).

Setting the data processing schedule

  • As a rule data processing can take a long time (from several minutes to several hours depending on the entered data volume and computer performance), and its execution can slow down user's work. So it is recommended to start data processing during user lowest activity period.
  • In client-server mode you can set up a Delayed database update scheduled job. Click the Configure schedule to set up the Delayed database update scheduled job.
  • You can also configure the scheduled job from the scheduled and background jobs list using corresponding command of the Administration - Support and maintenance section.

Locking the check of scheduled job

  • In case the scheduled jobs are locked in a client-server mode, the delayed update will not be performed.
  • It is possible to control llocking by:
    • Using the Test scheduled jobs locking link;
    • Using server cluster administration console. See platform documentation for console details.
      • It is possible to launch a console using the Start menu by selecting the All applications-1C:Enterprise 8 - Additional section.
      • Select relevant central server, relevant cluster, and relevant information database to display a certain infobase properties.
      • To view database parameters, click the database name in a list with right mouse button.
      • Search for users and scheduled jobs locking in the database parameters list.
      • For more detailed information about the selected database llocking, select the Locking group in a console tree.
