A application administrator often needs to find out which events occurred in the application at certain moments of time or which actions were performed by any particular user. This can be ensured through the event log.

The event log may register main actions performed to modify data, execute regular operations, connect to/ exit the application, etc. The event log is adjusted in the Configurator.

The event log can be opened by the corresponding command in the Administration - Support and servicing section.

The following types of events may occur when working with the application - every such event is indicated in the event log by a special icon:

- Error
- Warning
- Information
- Information on unavailable fields
- Note
Every event is recorderd in a separate line of the event log.

The list includes:

  • Date, time - in addition to the date and time, an icon identifying event type is displayed
  • User - the full name of the user
  • Computer - the net name of the computer
  • Application - for example, Thin client
  • Session - the number of the session
  • Event - the type of the event
  • Data, Comment - identification of the application object: particular list item, document, etc.; in the comment the event summary is given
  • Metadata - the type of the application object and its name: the list, document kind, etc.

To adjust the number, visual display and relative position of columns in the event log, press More - Change form.

Updating event list

  • The event list is not a dynamic list. To view events included into the event log, open the log and clickRefresh current list or press F5

Number of events

  • The event log shows a fixed number of records displayed in the Display field (by default it is equal to 200). IN addition to preliminary defined values (200, 500, 1000 and 10000 events), you can enter any number of events to be saved

Filters

Using filters you can filter events in the event log.

  • In the Criticality quick selection field, select the Error or Warnings value to view specific types of events. By default, the event log shows All events
  • For quick setting the period for events review in the log, click Interval. To go to setting date intervals, you can double-click the content of the Date, Time column
  • Select data in one of the log columns, click Set filter by value in current column. By this the application will filter log events which contain these data. The same command can be found in the right-click menu
  • To filter events by one or several criteria, clickSet filtering. You can also follow the link to the Filter field. If any filters are set (including quick filters), the link takes the values of the filtering criteria, and if no filters are set, the link takes the value Not set. Filters can be set in the following ways:
    • in the Interval field choose the period, specify the Beginning of period and End of period
    • in the Importance list, using checkboxes choose one or several types of events (Error, Warning, Information, Note) by which events should be filtered. All checkboxes are selected by default
    • in the Events field, choose values from the list using checkboxes (all checkboxes are selected by default)
    • in the Users field, choose values from the list using checkboxes (this list specifies only users included in the event log. The checkboxes are cleared by default)
    • in the Applications field, choose values from the list using checkboxes (Configurator, Thin client, Background task. The checkboxes are cleared by default)
    • in the Computers field, choose the needed values from the list using checkboxes (The checkboxes are cleared by default)
    • in the Sessions field, list the needed numbers of sessions separated by ";"
    • in the Data fields group, you can choose one of the following filtering types to facilitate the search:
      • in the Metadata field, using checkboxes choose needed values from the list (the list is ordered by data types) for filtering: choose the needed name of the list, document, etc., for example, "Russian banks classifier guide"
      • in the Data field, you can select search elements from application lists:
        • select the Data type, for example, "Bank"
        • from the corresponding application list, for example, "Russian banks classifier", select a particular application object, for example, "Sberbank"
    • in the Data presentation field, input search data manually or copy the data from the list
    • In the Transaction fields group, choose the Transaction status from the list using checkboxes (No transaction, Fixed, Not completed, Canceled). All checkboxes are selected by default. Filtering by the transaction Identifier field is also possible
    • in the Other fields group, you can select with checkboxes Working servers, Main IP-ports, Auxiliary IP-ports, and also perform filtering by the Comment field
  • After specifying fields for filtering, press OK. After that events will be filtered and the link will take the form of the filtering criteria list
  • Records satisfying the set criteria will be displayed in the amount not exceeding the value of the Display field
  • To disable all filters (including quick ones), click Remove filters or run the More - Remove filters command. This command can also be found in the right-click menu

Viewing event

  • To view detailed information on an event, select it in the list and click Open. This command can be found in the right-click menu
  • You can also view the event by double-clicking the required line

Viewing application objects

If the event is linked to a application object, you can open this object for viewing.

  • Select the required data in the Data, Comment column of the corresponding event and clickOpen data for viewing. This command can also be found in the right-click menu
  • You can also open data for viewing by double-clicking content of the Data, Comment column
