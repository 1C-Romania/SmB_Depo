Properties of a background job can be described as follows:

Basic

  • Name - a copy of the Name field for a scheduled job. Use the Execute now button on the scheduled job command bar to assign the "Manual start: <Scheduled job presentation>" value to the background job.
  • State - possible values are "Job completed", "Job completed with errors", "Job cancelled by user", "Job in progress".
  • Start date - date and time when the background job was started.
  • End date - date and time when the background job was finished (if the job is in progress, the values are empty: <>).
  • Execution location - name of the network computer without a domain on which the job was executed. If the scheduled jobs were processed in a file infobase, the name is either the server name or the user computer name.

Advanced

  • ID - it is useful only for an advanced administrator who knows the internal 1C language. It is required to examine non-standard issues of background job operation if any.
  • Key - value of the Key field specified in the scheduled job (when the background job was created) or another expressive value if the background job is software based and not created with the server planner.
  • Metadata method name - name of the configuration procedure which is being performed or was performed in the background job. Usually, it corresponds with the value of the Method name field of the relevant scheduled job.

Scheduled job

  • Name - one of the field values is selected (in sequence of search for entered information):
    • Name
    • Synonym
    • Name
  • ID - identifier of the scheduled job corresponding with the background job.

Message to user and error description

  • The field contains the text of messages and error descriptions that occurred during execution of the background job.
