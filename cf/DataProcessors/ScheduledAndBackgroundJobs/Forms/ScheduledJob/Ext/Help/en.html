Description of scheduled job properties.

Generally, scheduled jobs do not have to be changed. However, you can do it by clearing the Enabled check box. 

Sometimes you need to create two schedules for one scheduled job. Then you can add a new scheduled job with one of the predefined templates.

General

These are the essential properties for scheduled job setup. They are included in the supply just like all other properties and need to be changed only in exceptional cases.

  • Name - name of a scheduled job. It is usually empty. Instead, the application uses the synonym stated in the scheduled job template made with Designer. If no template synonym is available, the application uses the name.
  • Enabled - use this check box to enable/disable execution of a scheduled job. For example, when you want to disable a scheduled job for a while without editing the schedule.
  • Use the Schedule link to configure the scheduled job schedule.

Actions in case of errors

  • Repeat, times - number of attempts to execute a scheduled job in case of an error.
  • Repeat every, sec - a period (in seconds) between attempts to execute a scheduled job in case of an error.

Advanced

  • Predefined - Yes/No value. Manually defined scheduled jobs are hardly ever used.
  • Identifier - useful only for an advanced administrator who knows the embedded language. Required to examine irregular problems in the operation of scheduled jobs if any.
  • Key  - useful only for an advanced administrator who knows the embedded language. Required to fine-tune scheduled job execution and examine irregular problems in the operation of scheduled jobs if any. If the key is specified, then before starting a background job the system checks for other background jobs under the template that already works for the scheduled job instance.
  • User name  - name of the infobase user that will be used to execute the scheduled job. Select it from the Users list.

Metadata

  • Name - name of the scheduled job template defined in Designer.
  • Synonym - synonym of the scheduled job template defined in Designer.
  • Method name - name of the configuration procedure to be executed in the background job created under the scheduled job schedule.

Message to user and error description

  • The field contains a text of any messages and description of the errors during the last background job created under this scheduled job. The same content is available when you open the last background job of this scheduled job.
