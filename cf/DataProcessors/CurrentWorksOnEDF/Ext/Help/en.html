To-do lists on EDF data processor

Use this data processor to view all incomplete electronic document to-dos and complete them on the spot.
Operations with electronic documents are divided into groups and depend on the nature of the incomplete event: generating, processing (for outgoing and incoming documents), and control (waiting for actions for other participants).
Some groups have lists of (outgoing and incoming) packages waiting for the user's actions.

This data processor is a special workplace to work with the electronic documents which EDF process is not yet complete. On the left, there are groups of possible electronic document actions; on the right, there is a list of electronic documents requiring a certain action. There is a quick filter for the electronic documents in the form.

In the form, you can see the Send and receive command used to send all prepared electronic documents and receive the new ones.

In the All actions menu, when the Abort command is run, the selected items get the Aborted status. For example, with this action, the user can cancel an exchange (if it is not yet initiated, i.e. "ED status" = "Not generated") if the initial documents have been passed on paper.

Purposes of the groups in the list of available actions:

- Generate: a list of infobase documents requiring electronic documents. For the selected list items, you can use the Generate, sign and send and Generate new electronic document commands.

- Process: electronic documents requiring your actions, e.g. confirm or sign. For the selected list items, you can use the Confirm, sign, and send command.

- Confirm: electronic documents to be confirmed. For the selected list items, you can use the Confirm command.

- Sign: electronic documents to sign. For the selected list items, you can use the Sign and send command.

- Reconfirm: electronic documents rejected by the receiver and requiring amendments.

- Correct: sent electronic documents requiring correction or outgoing documents that could not be sent.

- Send: electronic document packages prepared for sending. For the selected list items, you can use the Send and Cancel command.

- Unpack: unpacked electronic document packages. For the selected list items, you can use the Unpack and Cancel commands.

- Controlled: sent electronic documents waiting for actions from other exchange participants, e.g. the receiver or the EDF operator.

- Recognize: electronic document packages, received but not recognized. For the selected list items, you can use the Cancel command.
