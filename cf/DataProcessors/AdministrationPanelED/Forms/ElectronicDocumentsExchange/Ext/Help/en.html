Electronic document exchange

Use this panel to enable and configure electronic document exchange between organizations and counterparties or direct exchange between banks.

Use the Electronic document exchange check box to use the electronic document mechanism in the accounting system and to customize other exchange parameters (for some options, digital signatures are required).
Digital signatures enable operations with cryptography in the accounting system. To use data encryption and digital signatures, the computer needs a special software encryption provider: e.g. CryptoPro CSP, ViPNet CSP or Microsoft CSP. To specify the cryptography settings, use the Cryptography setup form.

Select Direct electronic document exchange with banks to enable direct interaction with the bank software without installation of an individual Bank client module on the user computer. Direct exchange with bank requires digital signatures.
Electronic document usage agreements are intended to configure direct exchange with counterparties and to provide exchange through an EDF operator. Use Standard electronic document usage agreements to set up quick completion templates for direct exchange agreements with counterparties.
If the Postponed electronic documents sending check box is not selected, the package prepared for sending will be transferred to the recipient as soon as it is ready. If the check box is selected, then the application firstly collects the data to send,  forms a queue and sends the packages after the the separate user command Send and receive electronic documents  or within a predefined scheduled job.
