To change the attributes after data entry, you can select special settings using the Additional options command:

  • Change in transaction (check box is selected by default) - the application allows processing the list of items selected to be changed in transaction. If it is not possible to record one of the objects (for example, because that object is being edited by other user) during recording with the check box selected, then changes in all objects will not be saved.
  • Interrupt in case of error - if objects are not updated in transaction, then it is recommended to select this check box (this check box is selected by default). IN case object recording is not possible due to some reasons, the application will display the warning and offer either to continue updating the remaining objects or to interrupt processing in order to find reasons of the problem. If the check box is not selected and the object can not be recorded, then the application ignores this object and continues updating the remaining objects.

  • Process child items - this check box (selected by default) allows changing child items of the selected groups and parent objects without necessity to select check boxes for all items of groups and child objects. If the check box is cleared, the application processes the group instead of its child items.

Additional parameters for the administrator

The following additional option is provided during administrator work using the Attribute group update conmmand of Administration section - Support and service

  • Display service attributes and tabular sections - service attributes including service attributes of tabular sections are hidden by default. To display these attributes in the application object windows, it is necessary to select this check box. Service attribute editing can break the application operativity, therefore, it is recommended to use this option carefully and only for service purposes.

  • Parameter Process child items is not available in this mode.
