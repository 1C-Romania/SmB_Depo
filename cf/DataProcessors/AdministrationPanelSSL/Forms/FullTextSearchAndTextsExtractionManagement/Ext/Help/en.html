Full-text search is used to search the data in the application for content directories, documents and other objects in the application.
To start working with full-text search must include text search box data.
You will be able to manage full-text search IndexA, update and clean it.

Commands to manage the full-text search:

  • Update indexA
    		•  Updates the full-text search by updating to it the new or changed objects.
  • Clear indexA
    		•  deletions all the files of the full-text search.

     

    Automatic text extraction

    For the full-text search in the application it is necessary to extract the text from the files.
    In the client- server variant if one or more work processes of the server are running under Linux and the part or all of the clients are connected through a web client then to the infobase can be added the files the text of which is not extracted and, accordingly , these files can not be found by the full-text search of the content.
    In this case, on one of the client computers that are running under Microsoft Windows it is necessary to run thin or thick client  and to open in it the processor "Automatic text extraction". This processing will automatically extract text from files.
    By default the execution time interval is 60 seconds .


    In the file variant of work the texts are extracted automatically when loading the file to the application.

  • Edit the regulation task
    		•  Schedule and other parameters of the regulation task for the text extraction from the files.
  • Text exctraction
    		•  To run an automatic text extraction from the files on the work station under OS Windows.
