This panel is used for setting the data synchronization with other applications and shared work in the distributed database.

  • Data synchronization - select this checkbox in order to activate the Data synchronization application section and commands as follows:
    • Data synchronization - activates settings and performance of data synchronization with other applications (for example, with 1C:Enterprise Accounting 3.0), as well as shared work in the distributed database.
    • Warnings during data synchronization - activates viewing of warnings which arose during data synchronization (unposted documents, missing attributes, conflicts and data rejected by the date of prohibition).
    • Current database prefix - a prefix is added automatically in catalogs items codes and documents numbers to distinguish applications in which they were entered, and to avoid data duplication.
    • Change - by means of this command it is possible to assign database prefix or to change it if necessary.
    • Data import prohibition dates - activates prohibition to import the data of previous periods from other applications. See also: Data change prohibition dates in the Support and service subsection.

Servers cluster temp files directory of 1C:Enterprise

  • Path to a common shared directory for servers general temp files of 1C:Enterprise working in one cluster:
    • Under OS Windows - specify a path to temp files common directory for all server clusters which will be used for servers working under OS Windows.
    • Under OS Linux - a path to temp files common directory for all server clusters which will be used for servers working under OS Linux.
  • Temp files directory can be empty when in a cluster of 1C:Enterprise only one server or application works in file mode.
  • If all server clusters work under operating systems of one family (Windows/Linux), then the second field can be left empty.
  • The directory has to be available for reading and writing for all server clusters. Including all users of operating systems who start 1C:Enterprise servers.
  • UNC path to a directory has to be identical to all servers of one family - to do this, you need to connect to servers under one name.
  • The Linux temp files directory field can be used to specify a local path only, for example, /home/path.
  • It is necessary to make a link to UNC path to use it. For this purpose under administrator rights (root) make the following:
    • mkdir folder
      --- instead of folder write directory name. For example, in directory /home make mkdir shared - this command will created a directory shared with full path /home/shared.
    • mount -t cifs //server/share folder -o //domain/user, password=pwd
      - where //server/share - UNC path on other computer , folder - directory path on Linux computer, for example \home\shared, //domain/user - name of domain/user if necessary, pwd - user password (if necessary).
    • If name and password are not necessary, the command looks like:
      mount -t cifs //server/share folder
    • After that, specify a folder in the path to directory, for example, /home/shared.
