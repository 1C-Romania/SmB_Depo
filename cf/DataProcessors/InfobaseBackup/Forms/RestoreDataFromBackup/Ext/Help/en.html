It enables you to restore data from backup.

To open it, in the Administration - Support and service - Backup and recovery section, run the Recovery from backup command.

You can restore data from a previously created backup at any given time. You should take into account that all the data in the restored infobase will have relevance at time of backup creation. Data entered to the application after backup creation will not be present in the restored IB.

Data recovery

  • Using button Select specify path to the file with backup, from which infobase will be restored (required field).
  • In this assistant on data restoring you can use only .zip files containing one file - file of 1cv8.cd infobase. It's a good idea to use archives created by the backup creation assistant.
  • The application reports a warning stating that there are other users who are currently working with the infobase. Click the hyperlink to View all active sessions.
  • Click Restore data. If there are active sessions, they will be disconnected from the infobase. Then application will perform data recovery.

Database login in case of data recovery failure

  • Important: To log on to the infobase if data is not recovered, run 1C:Enterprise with the following parameters: ENTERPRISE /F <path to infobase> / CAllowUsersWork /UC Backup.
