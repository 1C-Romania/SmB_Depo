With the help of this toolbox you can configure the use of security profiles of the 1C:Enterprise servers cluster for the infobase. Security profiles are intended to control use of external resources by the application (relative to the 1C:Enterprise servers cluster).

It is located in the Administration - Support and servicing - Security section.

Security profiles use mode

Using an option button you can select a mode of using security profiles for the application:

  • No security profiles used for this infobase
  • The application does not inform the administrator of the need to add external resources into security profiles
    • If you set the use of security profiles in the servers cluster and do not change this setting, then any attempt to call external resources not authorized in the servers cluster would lead to errors in application operation
    • If in a security profile the Full access to external reports and data processors checkbox is cleared, the application start will be impossible
  • Security profiles for this infobase are configured by the administrator in the cluster console
  • The application does not inform the administrator of the need to add external resources into security profiles
  • The administrator configures authorizations in security profiles using the cluster console
    • Whereas the administrator may study the list of authorizations that have to be made in a security profile using a report which is opened when following the Required authorizations link in the toolbox
    • For an infobase profile, the Safe mode profile checkbox has to be selected
    • In the Infobase security profile field, specify the name of the security profile set for the infobase
    • If the Full access to external reports and data processor checkbox is cleared in the security profile, you have to select the Can be used as safe mode profile checkbox for it and specify the name of this security profile in the Infobase security profile field in this toolbox (otherwise application start will be impossible)
    • To offer to set up authorizations automatically in security profiles when necessary (directly from the application)
    • When switching to this mode, a new security profile will be created (and assigned for the infobase) in which all external resources required for correct operation of the application will be authorized
    • The application itself manages authorizations in security profiles
    • The application itself manages profiles
    • If when editing any settings of the application it becomes necessary to provide access to a new external resource, the administrator views a dialog with the list of authorizations
    • If the administrator agrees - he enters authentication parameters in the cluster and clicks Next
    • Authorisations in security profiles are provided automatically

            Required authorizations

            The command is not accessible if Security profiles use mode is set to For this infobase no security profiles is used.

            • Click Required authorizations to display the report on external resources which shall be authorized in security profiles for correct operation of the application

            Infobase security profile

            The field is not available for editing if Security profiles use mode is set to To offer to set up authorizations automatically in security profiles when necessary (directly from the application).

            If Security profiles use mode is set to Security profiles for this infobase are configured by the administrator in the cluster console and the Full access to external reports and data processors checkbox is cleared in a security profile, then you should select the Can be used as safe mode profile checkbox for the security profile and specify the name of this security profile in this field (otherwise application start will be impossible).

            • In the Infobase security profile field, enter the name the security profile used (in the 1C:Enterprise servers cluster)

            Restorring security profiles settings

            The command is available only if the Security profiles use mode is set to To offer to set up authorizations automatically in security profiles when necessary (directly from the application).

            It completely re-establishes security profiles for the infobase with account for external resources that may be required for application operation and taking into account current settings of the application. It can be used, for example, after transfer of the infobase to another servers cluster.

            • Click Restore security profiles settings
