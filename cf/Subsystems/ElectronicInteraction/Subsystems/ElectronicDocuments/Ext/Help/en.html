Subsystem is capable of exchanging of electronic documents (ED) with counterparties: exchanging of the product catalogs, price lists , orders, TORG-12, acts , invoices for payment etc., and also perform a direct exchange of payment orders and extracts with the bank.
For electronic documents there is the possibility of an optional (not obligatory) use of electronic digital print full name (EDS) , which gives the exchange between the parties a legal significance and eliminates the use of papers. An exception is the exchange with the banks - the use of digital print full name there is obligatory .

In a direct exchange between the parties through the open communication channels there is an additional option to encrypt the data which is being transmitted.

The application automates the exchange of electronic documents for a single transactions through exporting and importing the data through the electronic document without presets as well as the work with electronic documents for repeated transactions, where the task of the initial setup is obligatory.

Setting the exchange of electronic documents

How to start using the opportunities of electronic documents exchange in the application?

The application provides an easy opportunity to import and export the data through an electronic document without any prior setup , namely , data import through the document of information database on a computer disk or sending such a document via email to the counterparty in the form of an electronic document which is intended only for the automation of data input on the receiving side .
On the receiver side there is the possibility by an easy file selection to export the data to an existing document or to Creating a database on their basis.
This option of exchange allows you to automate the one-time transactions.

In order to start using the functional exchange of electronic documents for repeated transactions you must enable this feature separately in the general application settings and make preset of agreements on the use of electronic documents.

The application supports the following options for the exchange of electronic documents:

  • Easy import / export of the data through the file ;
  • Exchange through the electronic document management opertor (EDM operator);
  • Direct exchange between the counterparties ;
  • Exchange between organizations within the same company ;
  • Direct exchange with the banks.

    Certain settings in the application provide the opportunities for the use of:

    • electronic documents exchange when transferring the goods between organizations within a company;
    • direct interaction with the banks;
    • delayed sendings of electronic documents;
    • electronic digital print full name (EDS) and / or encryption

    Group basis is changed automatically depending on the operations performed with the electronic document.

    How to see which electronic documents are associated with an accounting system document?

    All electronic documents associated with the accounting system document are available in the list which can be opened using the command "Electronic documents".
    Through this list it is possible to open any electronic document for viewing, as well as right here to perform the necessary actions with it.

    Where are Saved the received electronic documents?

    After all the electronic documents are received and unpacked they go to the archive of incoming electronic documents.
    The archive form opens by the command "Archive of electronic documents".

    What does the form "Archive of electronic documents" contains and which actions can be performed in it?
    • The form contains the archive of electronic documents with tabs "Incoming" and "Outgoing" as well as the lists of packets of electronic documents.
    • In the list of electronic documents archive it is possible to set a selection by a person in charge, type and status of electronic documents.
    • In the list it is possible to perform the standard commands of the list, open the form of the electronic document for viewing.
    • On the tab "Unsent packages" it is possible to send a particular package out of the queue by the command "Send the selected packages".
    • On the tab "Unpacked packages" it is possible to unpack a particular package by the command "Unpack the selected packages" (such a need arises if data encryption is used under the direct exchange).
    • The tab "All Packages" contains all the received or generated packages of electronic documents. The data files of the already processed packages (delivered to the recipient or successfully unpacked) can be removed through the command "All actions" - "deletion the processed packages files".

    How to view the content of the electronic documents package?

    The content of electronic documents package can be viewed by saving the package file to the disk .
    For a group import of the files of ED packages it is necessary on the tab "All Packages" to choose the necessary packages in the list and call the command "All actions" - "Save ED packages to the disk".
    A similar command "Save package to disk" is available in the form of the document itself "Package of electronic documents".

    Is it possible to "reassociate" the incoming electronic document with another accounting system document?

    In the form of an incoming electronic document using the command "All actions" - "Select the document of reflection in accounting" it is possible to change the connection of an electronic document with an accounting system document.
    This command does not refill the accounting system document according to the electronic document (for this purpose it is necessary to use the appropriate command in the form of electronic document).

    Is it possible while exchanging with the counterparties in the application to confirm to the sender of an electronic document that the data is received?

    If on the phase of the setup of direct exchange agreement with the counterparties there was set a demand of a response print full name (set a sign "To sign" on the tab "Incoming Documents" next to the desired type of electronic document), then in the viewing form of an electronic document it is possible to set the EDS for the incoming electronic document by the command "Sign and send confirmation" or send it to the person in charge for the print full name by the command "Approve an electronic document".

    Important. While exchanging the primary documents in the electronic form of the FTS format that requires a response title for the incoming electronic document, after the command "Approve the electronic document" is executed the application Creatings a new electronic document of the necessary title and offers to sign by EDS.
