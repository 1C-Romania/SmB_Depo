The report is used to view information of all application users or external users including login settings (infobase user properties).

The report is necessary when you need to make group analysis of login settings (login name, kinds of authentication, etc.).

You can open the report from the list of Users (External users) by running the More - User (external user) information command.

The report contains the following information:

  • User full name
  • Application login is allowed - the field value can be "Yes" or empty
  • Name (for login)
  • Standard authentication - the field value can be "Yes" or empty
  • Authentication by OS - the field value can be "Yes" or empty
  • OS user - name used to login to OS
  • OpenID authentication - the field value can be "Yes" or empty
  • Start mode

User filters

  • Using Settings you can open report settings. To filter users basing on one or several options, select corresponding check boxes.
  • You can select users based on the following parameters:
    • Not valid
    • Marked to be deleted
    • Without infobase user
    • Application login is allowed
    • Standard authentication
    • Authentication by OS
    • OpenID authentication

Report variants

  • Using Report variants you can open the list of other available report variants. There are three report variants:
    • Information about users - you can open it from the list of Users
    • External user information - you can open it from the list of External users
    • Information of users and external users - the summary report of all application users where the This is external user field (changes to the "Yes" value or remains empty) enables you to select and filter external users
