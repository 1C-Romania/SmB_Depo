It is intended for indication of access parameters for servers cluster.

You can open it from the User locking window by clicking link Enable management of scheduled jobs locking.

Parameters required to perform various administrative actions with client-server infobase:

  • name and password of infobase Administrator to lock scheduled and background jobs on his or her behalf. IN the simplest case it can be your name and password which you use to access the application. For security reasons it is recommended to create a separate user with administrative rights. It should be noted that name and password are stored in the base itself and can be read if an attacker gains full access to the base.
  • Cluster administration parameters:
    • name and password of a servers cluster administrator on the part of whom forced termination of users sessions is executed. If is not specified if no administrator is configured in the servers cluster;
    • Port numbers for access to the cluster of servers and server agent if they are not standard. As a rule, the port number of server agent differs from the port of servers cluster per unit, for example: 3040 and 3041.
    • Connection to the servers cluster - choose an option from the list:
      • COM-connection with server agent
      • Through administration server (ras).
    • Address - for example, "demo".
    • Port - for example, "3040".
