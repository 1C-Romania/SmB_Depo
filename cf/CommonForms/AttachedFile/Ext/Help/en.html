It is intended for viewing information about the attached file.

It is opened from the list of attached files by means of the More - Open card command (F2 button).

File info

  • Description - the full name of the file. It is automatically filled in by the application when the file is attached and the file card is created, it can be changed
  • Details - file details to be entered
  • Author - the user who added the file to the base
  • Created - the date when the file was added to the base
  • Edited - the last user who edited the file
  • Changed - the date of changing file data (local time)
  • Location - the name and date of the document to which the file is attached. You can view the document
  • Size - the size of file data
  • Extension - the extension (format) of the file

Editing attached file

  • In thin client, editing table documents files (.mxl format) is done in the imbedded form of a table documents editor. IN web client, it is necessary to import an application for editing these files. Here you can find details about editing table documents
  • For editing files of other formats you should install the related applications on the computer, for example, for editing Microsoft Word files you should have this application installed
  • The View command opens the file using an external application for viewing or editing, depending on whether it is marked as being locked by other user for editing or not
  • The More - Edit command opens the file in editing mode and also locks the file in the application. The locked file becomes unavailable for editing by other user of the application and is displayed in green in the list
  • The More - Finish editing command unlocks the file, saves it in the application and removes the edit locking mark. When you finish editing, save the file and close the external application
  • The More - Cancel editinging command unlocks the file without saving changes of the file made in the application and removes the edit locking mark. This command can be useful if you need to invalidate file editing results (for example, to cancel unwanted changes or errors entered to the file)

Saving file

  • With the help of the More - Save as command, you can save the file to the specified directory on the hard or network drive
  • Using the corresponding command of the More menu, you can Open file directory in Windows Explorer

Updating file

  • With the help of the More - Update from file on hard drive command you can update the file from the original file on the drive (based on which the file was initially created in the application)

Digital signature and encryption

When you use a digital signature, the More - Digital signature and encryption drop-down menu becomes available, which contains the following commands for working with digital signatures:

  • Sign - creates a digital signature for the file. Before creating a digital signature, the application suggests choosing a digital signature certificate from those available on the computer. You can choose the certificate also during application settings customization. A file being signed becomes unavailable for editing and is displayed in lists with the  icon
  • Add digital signature from file - adds signature to the file. You can use this command, for example if the signature (signature file) was obtained later than the signed file was
  • Save with digital signature - saves signature to the file on the hard drive. The command can be used, for example if you need to pass a signed file to another person
  • Encrypt - starts file encryption. Unlike signing, several certificates may be used, for example if you need to encrypt a file for several users. Encrypted files are displayed in lists with the icon

  • Decrypt - initiates file decryption. For decryption you need to enter the password for the private key

When using digital signatures and encryption, the following tabs are available :

Digital signatures

The tab includes information about the file digital signature:

  • Signature author - the user who signed the file
  • Signature date - the date of signing the file
  • Comment - additional signature information to be entered 
  • Status - the signature validity status

With the help of the related commands you can:

  • Sign a file
  • Open the signature - to open the window with basic properties of the digital signature certificate for the selected file: to whom it was issued and by who, purpose and validity period of the certificate on the basis of which the signature was created
  • Check if someone changed the signed file using one or several digital signature certificates. For this purpose, select one or several signatures. If the digital signature of the stored file does not correspond to any digital signature certificate, such digital signature after the checking is displayed in red in the list and has the Not valid satus in the status column
  • Check all - to check all signatures
  • Save the digital signature to the file on the hard drive
  • Remove the digital signature from the file

Allowed to decrypt

  • The tab includes information on file encryption
  • In the list, this tab displays the certificates used for file encryption. By double clicking or using the related command, you can Open digital signature certificate and view its main properties: to whom it was issued and by who, its purpose and validity period
