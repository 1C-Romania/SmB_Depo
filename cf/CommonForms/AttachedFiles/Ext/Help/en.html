It is intended for working with files attached to a application object.

It is opened using the Attached files command on the the application object navigation bar.

The list includes:

  •  - availability of a digital signature or encryption
  • Description - the full name of the file. An icon next to the description specifies the type of the file. In the column, there may also be file details displayed if any
  • Author - the user who added the file
  • Change date - the date of changing of file data (local time)

With the use of the  More - Additionally - Change form command, you may add the following information:

  • Date of creation - the date and time when the file was imported to the application
  • Edited by - the user who occupied the file for editing, etc. 

Adding new file

  • You can Create a new file from a file existing on the hard drive
  • When you select a file on the drive, the application opens the Attached file window which describes properties of the file
  • The file is saved in a special directory. The application creates a separate folder for every its object
  • In some lists, items which have attached files have the  icon

Viewing file card

  • Click More - Open card (press F2) to view the properties of the file selected in the list by opening the Attached file window, or you may use the corresponding shortcut menu command

Editing attached files

  • In thin client, editing table documents files (.mxl format) is done in the imbedded form of a table documents editor. IN web client, it is necessary to import the application for editing these files. Here you can find details about editing table documents
  • For editing files of other formats, you should install the related applications on the computer, for example, for editing Microsoft Word files, you should have this application installed
  • The View command opens a file using an external application for viewing or editing, depending on whether the file is marked as being locked by other user for editing or not
  • The Edit command opens a file in the editing mode and locks this file in the application. The locked file is unavailable for editing by other user of the application and is marked in green in the list
  • When you finish editing, save the file and close the external application. Click  Finish editing, this action releases the file, saves it in the application and removes the edit locking mark
  • The More - Cancel editinging command unlocks the file without saving changes of the file made in the application and removes the edit locking mark. This command can be useful if you need to invalidate file editing results (for example, to cancel unwanted changes or errors entered to the file)

Saving file

  • With the help of the More - Save as command, you can save a file to the specified directory on the hard or network drive
  • You can also use the corresponding command of the right-click menu
  • Using the corresponding command of the More menu or of the right-click menu, you can Open file directory in Windows Explorer

Updating file

  • With the help of the More - Update from file on hard drive command, you can update a file from the original file on the hard drive (based on which the file was initially created in the application)
  • You can also use the corresponding command of the right-click menu

Digital signature and encryption

By clicking the Digital signature and encryption in the drop-down menu, you can use commands intended for working with files digital signatures:

  • Sign - creates a DS for a file. Before creating a DS, the application offers selecting a DS certificate among certificates available on the computer. You can choose a certificate also during application settings customization. A file being signed becomes unavailable for editing and is displayed in lists with the  icon
  • Save with digital signature - saves signature to a file on the hard drive. You can use this command, for example if you need to pass a signed file to another person
  • Add digital signature from file - adds signature to a file. You can use this command, for example if the signature (signature file) was received later than the signed file was 
  • Encrypt - a file is encrypted upon execution of this command. Unlike signing, several certificates may be used, for example if you need to encrypt a file for several users. Encrypted files are displayed in lists with the icon
  • Decrypt - a file is decrypted upon execution of this command. For decryption you need to enter the password for the private key
