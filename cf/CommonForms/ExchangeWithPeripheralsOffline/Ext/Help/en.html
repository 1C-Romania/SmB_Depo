How to clear the CR from the items which were loaded to it some time ago, for example, according to the old rules of exchange ?

To clear the list of items in the CR there is a command of the list  "All actions" - "Clear". And using the command "All actions" - "Restart" it is possible to clear the CR from the "old" list of items to load a "new" one.

How to export the items to the CR?

The info about the items is exported by the command "Export goods". The info is exported to the file which is specified for this cash register in the setting parameters of the trade equipment (field "Goods base").  Before the first exporting all the items which satisfy the exporting rules are marked as the changed ("Attention" icon). IN subsequent exporting to the list for exporting are ready only that goods according to which were changed the parameters: newly registered items, items with the changed price, barcode. IN this case in the exchange list with the "Attention" icon are marked also the cash registers the items of which have been changed.

How to open the rule of exchange assigned for the CR from the exchnage list (workplace)?

To open a rule of exchange from the exchange list is possible using the command "Open a rule".

How to assign for the several CRs the one rule of exchange?

To assign the one rule of exchange for several CRs it is necessary in the exchange list to select such CRs and use the following command of the list: "Set rule for selected".

And after in the opened  list choose the rule you need.

How to load the sales report generated by the CR after session is closed?

After the cash register offline session is closed and when generating the sales report file it is possible to load it into the application. The loading of the report file is performed in the form of the workplace using the command "Import sales report". After the command is done all the info about the sold items is fixed in "Report On Retail sales" document.

Is it possible to set up an automatic exchange of the application with the cash register?

Using the command "Automatic exchange" it is possible to set the interval through which the application will perform an exchange (loading and exporting) in automatic mode.
