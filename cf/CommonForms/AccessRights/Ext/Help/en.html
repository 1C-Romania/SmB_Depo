It enables you to view and set the access groups of a user (external user).

You can open it by running the User rights command in the navigation bar of a user (external user).

Every user can be assigned one or multiple roles based on profiles and access groups. Access roles determine user's personal rights settings.

Access groups define which actions on the application data are permitted to the group members. Access groups usually correspond to certain job responsibilities (or activities) of the application users. The user can be a member of one or multiple access groups at a time. The access groups define the user's personal access rights settings.

The following users can configure settings:

  • Administrators - for all application users.
  • Account owners of user lists and external user lists - for all users from the relevant list except for the access groups with administrative rights.
  • Account owners of the access group members - only for users of their access group.

In the list, you can see the following tabs:

  • Access groups - commands to set user rights and access groups the user belongs to.
    • The list displays the following data:
      • Name - an access group name.
      • Profile - a name of the access group profile.
      • Account owner - a responsible user.
      • Details - you can see the selected access group details in the lower part of the list.
  • Allowed actions (roles) - view permitted actions of a user (external user). User roles include all roles of the access groups the user belongs to.
    • You can arrange a list of roles in alphabetical order or By subsystems by clicking a corresponding button.

      Setting user rights

      • Click Add to group to add a user to any access group.
      • Click one of the access groups in the list, and then click Select. You can also double-click the group you need.
    • Click Exclude from group to delete the selected access group from the user access groups.
    • You can click Edit group to change the settings of the access group you selected.

        Report on user rights

        • To see a complete list of user rights, including the application objects and user (or user group) rights to them, run analytical report Report on user rights.
        • Click Report on user rights.
