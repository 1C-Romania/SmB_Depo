Combining Small Business, ver. 1.4 and Retail, ver. 3.1 applications

Advantages of combining Small Business and Retail applications

Combining Small Business and Retail applications allows automating business processes for retail outlets (stores) working within distributed retail network of trade enterprise. Small Business application is used as the management system installed in company's head office and Retail application is used to automate business processes of retail sales directly in stores.

Retail network can comprise any number of stores effecting retail sales and bulk sales of products. When communicating with the head office a store summarizes all data of the trade activity. IN other words data of goods flow inside the store and detailed data of retail sales are not transferred to Small Business application. Retail sales results are consolidated in Retail reports including cash in hand and are imported to Small Business.

Retail application installed in stores can share data with Small Business application installed in the head office directly or using Retail master node that combines several stores. Such network topology reduces load on the head office servers and enhances reliability of data distribution throughout the network using Retail internal communications. Retail master nodes provide an option to monitor leftovers in subordinated stores and to transfer the goods promptly between the stores.

Abbreviations used in the item:

  • SB - Small Business application
  • RS - Retail sales application

How to combine Small Business and Retail applications

When the data exchange is configured, work in both application is carried out concurrently. IN other words the Heads of Small Business application enter data of new products and services, prepare documents connected with products supply to stores (stock transfer).

Retail database users receive data on changes in the catalog data, prepare documents associated with the retail sales, retail price fixing and marketing operations. Data of business activity in retail network is transferred to Small Business application according to configured data exchange schedule.

When combining Small Business and Retail applications it is recommended to use centralized control and keeping regulatory documentation in Small Business application except when such control is not possible. For example, keeping records for gift certificates.

Restrictions and peculiarities of combining Small Business and Retail applications

In Small Business application the currency of records keeping shall be national currency because operations with foreign currency is not supported in Retail application.

To be used as a store in retail network for data exchange purposes, warehouses in Small Business application shall be described with the following values of attributes: Type of warehouse - Retail.

Data exchange settings

Data exchange between the applications is carried out on documents and reference information level. Documents and reference information are communicated in both directions.

Automatic data synchronization in both applications is carried out at initial setting of data exchange. When creating data exchange there is an option to select either Small Business application or Retail application as a source. When selecting source application the following shall be taken into account:

  • If one of the bases is practically empty, then selection of the source fully depends on the user preferences. It is worth taking into account that data exchange is created faster in less filled base since the time is consumed by data export when creating an exchange.
  • If there is a notable volume of data in both applications at the time of data exchange creating, then it is recommended to use the application where common data are presented in detail and correctly according to the user as the source database.

If not all common data shall be recieved as contained in the source application, it is recommended to select an option of interactive data loading and import from the source only the data that shall be imported as saved in the source application during first data exchange. The rest data will be synchronized later, when all required information is moved from application-receiver to application-source.

Before data exchange all general data used for data synchronization shall be entered to Small Business and Retail applications: Companies, SB Structural unit, warehouses (RS stores), Banking registers.

 

Preliminary settings of Small Business and Retail applications

Objects involved in matching data of the applications shall be created in both applications in advance :

  • Companies;
  • Small Business warehouses corresponding to Retail stores;
  • Banking registers involved in retail sales (for SB cash register type is independent).

It is recommended to name items of catalogs identically for both applications, it will facilitate interactive matching of objects.

In both applications the tax system for companies and store warehouses shall be configured.

Important! Filters applied for importing

Principle of importing filters is that at the time of data exchange Creating new settings are to all data. If filter settings are changed after communication is Creatingd, they will be applied only for the data changed after new settings are saved. Therefore it is recommended to think carefully over filter settings when creating data communication.

 

Creation of data exchange in Small Business application.

 

The use of data exchange is defined in data synchronization settings (Administration section - Data synchronization command). IN data synchronization settings form select the Data synchronization check box, set Infobase prefix. IN the Administration section open the Data synchronization list, click Configure data synchronization, in drop-down menu select Create data synchronization with Retail application, ver. 3.1...

So, the first page of data exchange creating assistant is opened. To go to the next page of the assistant click Next, to return to the previous page click Back. Select Step 1. Configure new data synchronization. On the next page it is necessary to select location of the second infobase:

  • Connect to the second infobase directly;
  • Connect to the second infobase using standard communication links.

If connected directly, it is necessary to specify second infobase connection settings: location (at this computer or at computer in local network, or at 1C:Company server) and authentication parameters (User, Password).

If connected using standard communication links, configure message traffic between infobases at next three pages :

  • Data communication using Local or network catalog;

On the Configure data communication settings page enter a name of communication (arbitrary). Name shall be selected taking into account the fact that the communication setting will be named in the same way by default in Retail application. Click Change data importing principles, and then enter communication settings:

  • Indicate the Company contract date of documents importing start
  • Select required check boxes to filter data: by company, by warehouses. Select companies, warehouses used to filter importing data at corresponding tabs.

Save the restrictions defined and click OK. Go Next to verify specified data exchange settings. On last page of the assistant used to configure data exchange click Save to save communication setting file, select the Import data now check box, and then click Done.

Extended communication settings in Retail application

The use of data exchange is defined in data synchronization settings (Settings and administration section - Data Synchronization command). IN data synchronization settings form select the Apply data synchronization check box, set Infobase prefix. IN the Settings and administration section open the Data synchronization list, click Configure data synchronization, in drop-down menu select Small Business, ver. 1.5...

In the opened form of data exchange Creating assistant select Step 3. Continue configuring data synchronization, specify path to previously saved data exchange setting file created at data exchange setting in Retail application. Click Next and configure if necessary, message traffic between infobases at next two pages.

At the Configure settings of data synchronization page there is an option to change Infobase names. Click the Change data importing principles hyperlink and enter data exchange settings.

If you need to import documents from a certain date, set the Document importing start date limit when configuring communications.

Save set limits and click OK. Go Next. Check previously configured data synchronization settings. On the last page of data synchronization setting creating click Done. Synchronization setup is completed.
