With the Event log analysis report, you can track the application operation according to the data of the registration log. The report displays statistical information about the application functioning, operational errors and warnings, number of active users in different time periods, and duration of sheduled jobs. 

To open the report, click Administrator reports in the Administration - Support and service section.

Statistical information is designed only for diagnosing monitoring of the application functions and the server load. Do not use it to evaluate your personnel efficiency, calculate payroll, etc. 

Report variants

  • User activity analysis
  • User activity
  • Tracking registration log
  • Scheduled job duration

User activity analysis

  • Displays statistics of user activity (working hours, number of connections, number of added/edited documents/catalogs, number of operational errors and warnings, maximum number of simultaneously working users). For a better convenience, some information is displayed as charts.
  • Settings:
    • Select Users from the list (you can select all users or users groups).
    • Specify Period.

  • Advanced settings: Click Settings in the More menu. To use the advanced settings:

    • Use the check boxes to enable or disable display of a number of added/edited documents/catalogs.
    •  Select the chart type from the relevant list.

User activity

  • Displays detailed statistics concerning activity of a user (a list of created/edited documents/catalogs/business processes/tasks). For a better convenience, some statistical information is displayed as charts. You can use it as a transcription of a user activity analysis report variant for a specific user.
  • Settings:
    • Select User from the list.
    • Specify Period.

  • Advanced settings: Click Settings in the More menu. To use the advanced settings:

    • Use the check boxes: select Yes to Display the data about your business processes, tasks, catalogs, and documents, or No to go on without it. By default, the report displays data about catalogs and documents.

Tracking registration log

  • It is a list of critical entries (about errors and warnings) in the registration log. Use it to track issues during the application operation. Use the report for prompt troubleshooting of frequent issues without reading the registration log.
  • Settings: Specify Period.

Sheduled job duration

  • You can see the execution time and working duration of the scheduled job sessions on a specified day. Analyze the server workload related to scheduled jobs, find peak load intervals and relieve them.
  • Double-click a name of the scheduled job in the report to view the Scheduled job data report and find the statistic data of the selected scheduled job.
  • Settings:
    • Select Day to get a report on a part of this day.
    • Specify Period start - time of the day when the scheduled job monitoring period starts.

  • Advanced settings: Click Settings in the More menu. To use the advanced settings:

    • Specify Period end - time of the day when you want to have the results of the scheduled job monitoring.
    • Select Yes to Display background jobs; clear the check box or select No to go on without it.
    • Specify the Minimum scheduled job session duration(in sec); by default, it is 1.
    • Specify the Minimum quantity of simultaneously working scheduled jobs for period; by default, it is 2.
    • Select the check box to Hide scheduled jobs.
