Designed to set up data synchronization and store existing data synchronization settings.

You can open the Data synchronization window in the Desktop or use the relevant command in Administration - Data synchronization settings. If there is no or only one setting, the window looks like a panel. IN this case, the More menu is hidden and all necessary commands are displayed in the panel. The window looks like a list if there are two or more synchronization settings.

 The window displays:

  •  - data synchronization is scheduled. The same mark is used in this column for the synchronizations performed automatically according to a predefined script.
  • Application - a name of the application for which a given synchronization is customized.
  • Infobase - a name of the infobase to synchronize with.

  • You also see a synchronization state (time since last synchronization in minutes, Now, Never). The  sign indicates failed synchronizations, the  sign indicates warnings:

    • Data sent - last time when data was sent.

    • Data received - last time when data was received.
  • At the bottom of the list, you see an Infobase prefix.

Customizing data synchronization

  • To customize new data synchronization with another application, select the relevant command in the Set up data synchronization submenu. It will open data synchronization settings.

Viewing settings

  • To get accounting information about the synchronized applications and get recommendations about data synchronization setup, click More - More about data synchronization with this application.
  • You can find a similar command in the context menu.

Data synchronization

  • Select the necessary setting, click  Synchronize to start automatic data synchronization.
  • You can also double-click the setting. Click Yes to confirm your selection.

Manual data synchronization

  • To synchronize data manually, select a setting in the list and click  Synchronize data with additional parameters. New data from another application will be loaded first, then data will loaded back.
  • Use the Synchronized data with additional parameters command to synchronize data after setting up similar data mapping. This procedure is useful when similar data has been added to both applications independently. To prevent most of such situations, you can set up frequent scheduled data synchronization.

Scheduled synchronization

  • Use the More - Data synchronization scripts command to Set up data synchronization according to a predefined schedule. IN this case, in client server mode, data will be synchronized in the background within the Data synchronization scheduled job; in file mode, you will have to start an additional session to perform scheduled jobs.
  • You can use a context menu to find the command.

Viewing warnings

  • Follow the  Warnings link (with the number of warnings within brackets) to learn more about warnings related to data synchronization.

Showing failed only

  • Click Failed only to show only failed synchronization attempts or attempts with warnings.

Viewing registration log

  • Use the More - Data sending events or Data receipt events menu commands to learn details of data synchronization process from the application registration log filtered by synchronization events and view each event.

Using synchronization rules from external file

  • To replace the standard object conversion rules and object registration rules with rules from an external file:
    •  open the data synchronization which rules you want to change.
    • Run the More - Export rules command.
    • In the object conversion/registration rules window, switch to Export from file on computer and click Export to select a rules file on the computer or in a network folder.
    • Check that you are exporting correct rules compatible with your versions of the source and receiver application. 
    • Click Save and close. The exported rules will be applied immediately.

Deleting synchronization setting

  • Select a setting, click More -  Delete synchronization setting.
  • Warning! The setting will be deleted permanently.

Restoring infobase from backup

  • When you restore an infobase from a backup, it is recommended to:
    • Temporarily disable automatic data  synchronization (clear the Automatic scheduled check box in all data synchronization scripts).
    • Cancel sending data prepared for other applications (for each data synchronization setting, run the More – Sent data content command; in the opened Changes registration window, in the object tree, select the root and run the Cancel registration – All selected subtypes command).
    • Resend the data entered into other applications since the last backup of this infobase - from other applications to this infobase.
    • Restore automatic data synchronization.
  • If considerable part of data is lost, it is recommended to delete all the data  synchronization settings and recustomize synchronization. Otherwise, you may lose all data in the synchronized applications because you send them outdated data from the restored infobase.
