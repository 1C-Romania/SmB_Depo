Designed to specify parameters of the file and client server base when creating an initial DIB image. When configuring a DIB synchronization (external files), specify where the initial DIB image will be created. If files are saved in volumes, you will have to specify the entire path to the file with archive (.zip) with all files in volumes.

Open it with the  Create initial image command from the Data synchronization creation assistant when you configure DIB synchronization (external files). In the Data synchronization list, you can use the  button to open DIB synchronization setup (external files), and click  Create initial image of subordinate node of distributed infobase.

Infobase parameters

Select where to create the initial image of the infobase:

  • On this computer or on a local network computer.
    • Define the Infobase parameters:
    • In the Language (Country) field, select the necessary value in the list. By default, you use parameters defined when creating the data synchronization.
    • Define the Full name of file base:
      • For 1C:Enterprise server under Microsoft Windows -  use the  button to specify the path to the folder for the file.

or

      • For 1C:Enterprise server under Linux - write a path to the folder for the initial image file.
  • On 1C:Enterprise server.
    • Define the Infobase parameters:
      • Define 1C:Enterprise server
      • Enter Infobase name in cluster
      • In the Secure connection field, select a value from the list (Off, Connecting, Permanent)
      • Select from the DBMS type list (Microcoft SQL Server, PostgreSQL, IBM DB2, Oracle Database)
      • Specify Database server
      • Enter Database name
      • Enter Database user and User password
      • Select Date shift (default value = 0)
      • In the Language (Country) field, select a value from the list. By default, it is the parameter defined when creating the data synchronization.
      • If necessary, select the Lock scheduled jobs (by default, it is cleared).

Path to volume files archive

  • Specify Full name of volume files archive:
    • For 1C:Enterprise server under Microsoft Windows - use the  button to specify the path to the directory

or

    • For 1C:Enterprise server under Linux - enter a path to the folder for the volume files archive.

To proceed, click Create initial image.

In a while, you will see the message about successful creation of an initial image of the subordinate node of the distributed infobase. Click OK to return to the application.

To add the volume files in the branch office base, at a first start, specify the path to the archive file with files in volumes, and run the Place files of initial image command in the action panel of the Organizer section.
