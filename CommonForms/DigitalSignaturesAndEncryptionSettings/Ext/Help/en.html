Adjusting parameters for use of means of digital signing or encryption (cryptography).

It is opened by the Digital signature and encryption settings command in the Administration - General settings section.

To use digital signatures and encryption, you have to set a dedicated external application (encryption provider).

In the Administration - General settings section, select the Electronic signature checkbox to be able to sign data, and the Encryption checkbox to be able to use encryption. Upon that the corresponding application commands become available.

The digital signature is an electronic document attribute with which you can make sure that no changes occurred in the document since you signed it. To put a signature, you should have a digital signature key certificate. You can obtain the certificate in a certification center. For example, it could be the certification center of 1C Company http://v8.1c.ru/learn/. Only a certification center may provide a legal (qualified) certificate.

In a distributed infobase, most of the settings can be changed only in the master node.

Certificates

  • The application displays all certificates used for data signing and encrypiton, as well as requests for new certificates
  • You may filter certificates in the Display field to see only your certificates. By default the application displays all certificates
  • You can filter certificate requests by their status in the Request status field ("Any", "Not finished", "Finished", "Sent", "Declined", "Completed - certificate not provided", "Completed")
  • You can Add a certificate from the list of certificates Set on computer. Or you can do the same during data signing or encryption
  • You can also use Add to prepare a Certificate request
  • You can display a certificate or certificate request if you double-click it. Or you can do the same through the Change command on the shortcut menu that appears on right-clicking or on the More menu

Applications

  • It contains the list of applications to be used in the infobase (including all nodes of the distributed infobase)
  • These applications do not have to be installed on all computers. Moreover if more than one application are installed on a single computer, they may conflict with each other
  • After the start, the list already includes two entries of the most popular applications in Russia - ViPNet CSP and CryptoPro CSP (KpиптoПpo CSP).
  • You can add entries of other applications (ready settings are supplied for Microsoft Enhanced CSP, ViPNet CSP, CryptoPro CSP (KpиптoПpo CSP), LISSI CSP (ЛИCCИ CSP), Signal-KOM CSP (Cигнал-KOМ CSP))
  • If you need to use a application other than the known ones, you should specify its name, type and algorithms as required by the related provider instructions
  • The Verification result column shows whether a application is installed on the computer or not , and if it is installed, it shows then whether correctly or not
  • Linux: Application path (<Application name corresponding to the selected string in the list>) is the field which you have to complete if your computer operates under Linux
  • On the bottom there is a link to Digital signature and encryption software instructions. If you have any questions, use this link to go to the corresponding section of 1C:ITS
  • Verify signatures and certificates on server - this checkbox is visible for back-end bases only. If you select this checkbox, then digital signatures and certificates will be verified on the server and you will not need to install the application on every computer where such operations are required
    • In the application entry you can go to the Application paths on Linux servers list where you need to indicate paths for the computers on which the 1C:Enterprise server operates under Linux

Additionally

  • Specify the Encrypted attachment file name suffixes when sending by email, running the Save as command and exporting a folder (by default the name suffix is .p7m)
  • Specify the print full name attachment file name suffix (by default the name suffix is .p7s)
  • Using the When saving data with digital signature to file radio button, specify application actions when the Save with digital signature command is run:
    • Save all signatures - all signatures are saved
    • Ask - the signatures to be saved are selected from the list
