The list provides more convenient mapping between the Designer infobase users and the Users and External user lists.

To open the list, use the More - Infobase users command from the user list menu (or the external user list).

When you save a user (or an external user), you automatically create a corresponding infobase user. Sometimes, the infobase user list data becomes being different from the user list data.  IN this case, you can use this list to:

  • Find "invalid" infobase users. 
    • The infobase users and the listed users are mapped automatically.
    • After mapping, infobase users get inactive if some application users have been added, edited or deleted with Designer.
    • Infobase users highlighted in red are not mapped with users in the application list.
  • Find data differences between the infobase user list and the lists of the application list users:
    • If an infobase user has been added with Designer (or in some non-standard software-based way), you can map this infobase user with a user in the list orDelete.
    • If an infobase user was deleted with Designer (or in some non-standard software-based way), but the list still contains a link to this user, you can open the user in the list and resave them to clear this link (or update the link by creating a new infobase user).
    • If a full name of the infobase user has been edited with Designer (or in some non-standard software-based way), you can open the user in the list and resave them to restore the full name of the infobase user.
  • View the list of all infobase users (also available in Designer).

The list displays:

  • Full name - a full name of the infobase user (when data is not mapped, may be different from the full user name in the Users list (External users).
  • Name (login) - when data is not mapped, may be different from the login in the Users list (External users).
  • 1C Authentication - Yes, when the check box is selected (otherwise, it is not filled in).
  • OpenID Authentication - Yes, when the check box is selected.
  • OS Authentication - Yes, when the check box is selected.

Additional fields (More - Change form):

  • OS user - a name used to log on to the OS and the application.
  • Mapped with external user - Yes, when this is an external application user.
  • Added in Designer - Yes or empty, when the field shows False.
  • Edited in Designer - Yes or empty, when the field shows False.
  • Deleted in Designer - Yes or empty, when the field shows False.

Viewing problem items or entire list

  • Select the Show only added, edited or deleted with Designer check box to view only unmapped items that require user intervention. When the check box is cleared, you see the entire infobase user list.

Mapping users

To map your lists, you can:

  • Map - manually map an infobase user with an existing application user:
    • Select User or External user to map, click OK. 
    • Select a user from the relevant list by double-clicking or finding them and clicking Select.
    • In the User (External user) window,  make necessary changes, click Save and close. After that, the lists mapping will be restored.
  • Map with new - you can manually map with an infobase user by manually entering a new user:
    • Select value User or External user, click OK. 
    • Enter a new application user (external user). After the data is saved, lists mapping will be restored.
  • Cancel mapping - you can cancel automatic mapping between infobase users and application list users to do it manually. Mapping cancellation is necessary very seldom - only after an incorrect mapping, e.g. when updating an infobase. IN other cases, cancellation is not recommended.

Viewing mapped user

  • Select an infobase user. Run the Go to user command in the More  menu or the right-click it to open the user (external user) mapped with the infobase user.
