To edit the user information it is necessary to have administration rights. IN other cases, it is possible to view the information about yourself as well as to change your password, application language and contact information.

User's full name is displayed in all the lists and reports. The check box Not valid is cleared by default for all users. But if the user for some reason no longer works with the application and at the same time it can not be removed because of the links to it from other objects in the application then it is recommended to set a tick next to Not valid. This user can not be selected, for example, as a task executive or responsible for the document and it is not displayed in the list of users .

To make the user work in the application, there must be set a tick next to Access to the info base is allowed and to specify the necessary properties : name, password and other authorization parameters. The login name in the application may differ from the full name ( e.g. "Ivanov" and full name - "Ivan Ivanov").

Here are recommendations how to Creating password.

Good password should contain
- not less than eight symbols,
- capital letters
- small letters,
- figures,
- symbols (underlining, brackets, etc.)
Good password should be a poorly understood expression.

Examples of good passwords:
“nj7{jhjibq*Gfhjkm,
F5”njnGhkmNj;t{HI.

It is undesirable that a password is the same as the username, consist only of the figures, contain clear words, contain alternating groups of symbols.

Examples of bad passwords:
Jason,
qwerty,
14445678,
144144144.
