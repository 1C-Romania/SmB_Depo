Designed to maintain email accounts that receive and send messages.

You can open the list with the relevant command in the Administration - Organizer section. 

The list includes:

  • Name of the account.
  • User name - a sender's name that may differ from the user name in the application.
  • Email address of the account in the name@server.net format. 

Passwords must be entered for the accounts that automatically send and receive messages. Configure the system email account to ensure correct operation of the application.

System email account

  • The application is delivered with a predefined account - the email account. By default, it is used for application notifications, e.g. to automatically send messages to the administrator whenever the application is updated. Before using these options, set up the system account.
  • Set up the system account with the relevant command in Settings and Administration - Organizer. You can also Edit it in the list.

Adding email account

  • When you try to save an account, the entered email details are checked automatically (a test message is sent). Accounts failing the test are not saved.
  • Click Create and fill in the required fields:
    • Specify the Email address of the account in the name@server.net format.
    • Enter the account Password. It is recommended to remember the password for the system accounts to allow sending messages from such an account automatically. Still, you should decide whether to store passwords in the infobase individually for each case.
    • Use the check boxes to define how to Use account:
      • To send mail
      • To receive mail
    • Define the Sender name (your or your company's name).
    • Select how the account will be created:
      • Define connection parameters automatically (recommended)
        • Click Create to set up and check the account automatically. Follow the instructions. If the setup failed, try to configure the parameters manually (see below). 
      • Configure parameters manually

    • Click Next to fill in the remaining parameters. For most mail servers, the recommended parameters will be defined automatically. Check them and try to save the account again.

Configuring email account

  • Click Edit and configure the settings.

Checking email account

  • Select the account in the list and click More - Check account to check the new account parameters. A test message will be sent to check the connection to the incoming mail server (if you use the account to receive mail).
  • If the account does not have a password, you are requested to specify it. Do it if necessary.
