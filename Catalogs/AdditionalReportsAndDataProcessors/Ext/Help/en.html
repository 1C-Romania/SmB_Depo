Designed to store additional reports/processors.

You can connect and use additional (external) reports/processors in the mode of 1C: Enterprise. Use them to expand the application functions without customizing it. It may help to promptly solve tasks with constant support of the application supplier.

Additional reports/processors are developed in the Designer mode, so you have to contact the developer. You can also get them from the Internet.

To use additional reports/processors, use the Print forms, reports, and processors command in the Administration section, select the relevant check box. Then get access to the Additional reports/processors command. By default, the check box is selected. 

The list shows:

  • Name- a full name of the report/processor.
  • Type- there are different kinds of additional reports/processors. They are defined by the developer. The kind defines functions.
  • Publication- visible of additional reports/processors is also highlighted with color. Red means additional reports/processors in debugging mode which are available only for the administrator. Gray means disabled additional reports/processors. The additional reports/processors marked for deletion also get inactive and are shown in gray and with a special sign.
  • Version- a version number for this processor (stated by the developer).
  • Responsible user - a user appointed to be responsible for this report/processor. They monitor states of additional report/processor, update versions, etc. (by default, it is "Administrator").

Use the More - Edit form command to add more columns:

  • Object name - includes a kind and other data. It is filled out by the developer and set by default.
  • Solutions compatibility mode- the application version for which the processor is used.
  • Comment - detailed information about the additional report/processor.
  • Uses variants storage- a property for additional reports. "Yes" if the additional report stores data in the application variants storage.
  • Use for list form - "Yes" if the command invoking an additional report/processor is on the lists windows;
  • Use for object form- "Yes" if the command invoking the additional report/processor is on the lists items windows.
  • Information- description of the additional report/processor, written by the developer.
  • File name - the extension and name of the additional report/processor file.
  • Safe mode- "Yes" if the additional report/processor is run in the safe mode. Normally, most processors are executed in the safe mode. It ensures that they do not contain an unreliable application code being executed on the server. Still some processors may require execution in unsafe mode. The application warns about it. Export such processors only from reliable sources (suppliers) or check them before export. For more information on the safe mode, see 1C: Enterprise platform documentation.

Adding additional report/processor

  • Click Create and select the file of external report/processor on your computer (specify a path on the disk) and configure its parameters. The set of data depends on the purpose of the report/processor. If a wrong file is selected, the application indicates an error, e.g. a message stating that the additional processor is designed for another application version. 

Grouping additional reports/processors

If the list is long, it is appropriate to group additional reports/processors by some criteria. To do it, enter the necessary quantity of groups:

  • Click Create group and enter the group name. The new group can be subordinate to another group: select the necessary group in the Parent field.
  • Drag necessary additional reports/processors to the group.

Filters

If the list contains a lot of additional reports/processors, it is appropriate to use filters:

  • By the Type field - select one of the additional report/processor kinds to filter, e.g. "Report". 
  • By the Publication field- select the necessary state to filter, e.g. "In use".
  • By the Type and Publication fields - combine the two filters in long lists.

Exporting from file

  • Use the Export from file command to update the processor in the internal application storage. Specify the path and click Open.

Importing to file

  • Use the More - Import to file command to load the additional report/processor from the storage to a file on the computer disk.  Specify the computer folder, and then click Save. The attachment file name is assigned automatically; you can rename it.

Kinds of additional reports/processors

There are several kinds of additional reports/processors :

  • Global reports/processors - used independently from the application data; also known as common. The commands for their use are in the application sections. Use the global reports/processors to import data to the application, update or clear the index of the full-text search, etc. They contain commands invoked only from the command interface sections. The commands do not depend on specific application objects. The list of available sections for commands placement is predefined by the developer and included in the file of additional report/processor; you can not change it.
  • Assigned additional reports/processors - designed for one or more specific documents or lists; the list of available assignations is predefined by the developer and included in the text of the additional report/processor; so when you configure them, you can select from this list. This type includes several additional reports/processors:
    • Report - an assigned additional report. Use it to generate a report for a specific application object.
    • Completing object - an assigned additional processor automatically completing or clearing one or more list items.
    • Print form - an assigned additional processor multiplying the document print form.
    • Creating linked objects - an assigned additional processor quickly filling in lists by copying items.
