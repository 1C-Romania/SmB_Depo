Encoding is a numeric pattern where each text character corresponds with a certain numeric value. Different languages usually consist of different sets of characters. Therefore, there are multiple encoding standards. What you see as a text is actually stored in the text file as numbers. The computer converts these numbers using an encoding standard and displays the final text.

The text file encoding is selected according to the relevant command when loading the file (loading the folder) from the drive to the file list. To load a file or a folder, you can use relevant commands of the context menu.

You can also Change encoding of an existing file in the embedded text editor by using the relevant command in the More menu.

Selecting text file encoding

  • Select an encoding from the list.
  • If there is no suitable encoding, clear the Show only main encodings check box.
    • A list of all known encodings will be generated.
    • Select the necessary encoding from the encoding list.
  • Click Select to save the setting.

If you open a file and see unknown characters, change the encoding. The main encodings for Russian text files are:

  • IBM866 (Cyrillic Dos)
  • KOI8-R (Cyrillic KOI8-R)
  • Windows-1251 (Cyrillic Windows)

General information

  • To avoid encoding-related issues and file decoding, you can save your files in the Unicode encoding. Unicode includes character sets of most languages used on computers.
