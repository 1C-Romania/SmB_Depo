Description of properties of a file storage volume.

Whenever you create a new volume, access to the catalog specified in the Full path field is automatically checked. If an error occurs, the relevant message appears. You can not change paths to the volume you have already created.

For each volume, specify:

  • Name - a full name of the volume for file storage
  • Code - it is completed automatically
  • Full path for 1C:Enterprise server under Microsoft Windows - a path to the catalog of the volume server under Microsoft Windows in the UNC format - \\Computer name\Folder name\File name\

or

    • Full path for 1C:Enterprise server under Linux - a path to the catalog of the volume server under Linux.
  • Sequence- enter an integer; by default, 1. A volume is filled out in ascending order until it is full. You can use this option, for example, for temporary disabling of the volume by increasing the filling order.
  • Limit, MB - you can use the maximum size of all the files in the volume (volume quota). If the total file size in the volume exceeds the quota, the successive volume is used. If no available volume has enough space to store a file, the information about it will be recorded in the event log. An error message will appear. If the quota is null, it is not checked when files are stored.
  • Current size, MB - it is populated automatically while filling out the volume and shows the volume occupancy.
  • Write detailed information about the volume on the Comment tab.

More about filling in full path field for Linux

  • If the 1C:Enterprise server works under Linux, you can specify only a local path to the file storage volume, e.g. /home/path. To use the UNC path, create a link to it. Use the administrator rights to do the following:
    • Run a command like mkdir folder, where specify a name of your directory instead of "folder". For example, in the /home directory run the mkdir shared command - a shared directory will be created with the full path /home/shared.
    • Run a command like mount -t cifs //server/share folder -o //domain/user, password=pwd, where
      //server/share - is the UNC path to another computer, folder - path to the directory on a Linux computer. For example, \home\shared, //domain/user - a name of the domain/user if necessary, pwd - a user password (if necessary).
    • If the name and password are not necessary, the command looks like this: mount -t cifs //server/share folder.
  • After it, specify folder, e.g. /home/shared.

Viewing file list in volume

  • In each volume, files are stored in subdirectories. An individual subdirectory is created for each file version creation date. All files are stored with their original names and extensions between them is the version number, e.g. "Order.2.txt".
  • To view a list of files in the volume, click Files in volume in the volume navigation panel.
