Description of key operations properties.

There are predetermined key operations in the application. It is possible to add new operations if necessary. Various application operations, for example, document posting, report generation etc. can be considered as key operations. It is necessary to fill key operations parameters when application starts to estimate application productivity (key operations parameters are not filled after application installation).

Key operations data

  • Name - is completed for the predetermined operations by default, it is necessary to re-enter for new operations. The field is mandatory;
  • Priority - specify the integer number for the priority of each operation, at that, the priority must not be repeated. The field is mandatory;
  • Target time - specify time in seconds. The field is mandatory;
  • MIN allowable level - select from the list, for example, Satisfactory.

The selected key operations parameters will be applied to measure the productivity.

Time measurements control

  • Time is measured for each key operation during application performance estimation. To view measurements history for each key operation, click Time measurements on navigation panel.
