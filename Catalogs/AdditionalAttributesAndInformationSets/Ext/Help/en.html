It allows you to configure sets of additional properties.

You can configure sets of additional properties for specific application objects. 

The storage method for an additional property depends on the property type:

  • Additional attributes - stored in the list item itself or in the document (they can be located in a special section or tab). They are created to implement the properties representing integral part of the object, entered when editing and available for editing by the users with access to the object with properties itself. 
  • Additional data - designed to store the data about a catalog item or a document when the data is not an integral part of this item or document. Generally, open for view/editing by the users without access to the object itself. They can be edited in a separate window that can be opened from the object window.

Different application objects may use additional attributes and data in various combinations: either additional attributes and data or additional attributes, or additional data.

To use additional properties, select the Additional attributes and data check box in Administration - General settings. After that you get access to the relevant commands. You can work with additional attributes (data) in the relevant lists opened with the Additional data command in the Administration - General settings section. 

You can open the set of additional attributes from the application objects for which they are set up. Use the More - Add or delete additional attributes command from the Additional data window of the application object.  

The Additional attributes (data) list is a table of two parts:

  • In the left column, there are Sets of additional attributes and data assigned for the catalog or documents shown as groups. If a catalog has groups, its Sets of additional attributes and data  correspond with the catalog groups.
  • In the right column, there is a table with the list of additional attributes (data) included in the selected set. Required attributes (data) appear bold in the list of additional attributes (data). The table shows: 
    • Additional attribute (data) - includes a list of the additional attributes (data) names.
    • Value type - specifies the type of additional attributes (data) and the number of values.

The predefined sets of additional attributes and data correspond with the application objects (catalogs or documents), where the additional attributes and data can be used. You can not add new sets to this list. Sets are automatically created when you create groups in relevant lists.

You can create a set of additional attributes and data for both all the list items and for some of them. In this case, the catalog is a group, and the group elements are catalog attributes or catalog attribute values (in this case, the number of the group elements will be equal to the number of the attribute values).

Assigning new additional attribute (data) to catalog or document

  • According to the configuration, you can use one of the command inputting new attributes (data):
    • Select the set that corresponds with this catalog or document, and then click Create. By default, an additional attribute (data) can be added to only one set. Fill in the required fields.
    • Select the set that corresponds with this catalog or document, and then click Add - New, the command is similar to Create.  Use this command to add the additional attribute (data) to one set. Fill in the required fields.

General sets of additional attributes and data

  • You can use Common sets for some catalogs. The additional  attributes and data entered into such a set, by default become common for all the items of these catalogs.
  • Select the Common set.
  • Click Create or Add - New. Fill in the required fields.

Adding from template

  • To use this option, select the Attributes and data with values list check box in the Administration - General settings section.
  • You can use previously entered additional  attributes and data from other sets, add attributes with a common values list for multiple one-type attributes (e.g. Door color from the Color attribute template). IN this case, the values list is taken from the sample attribute. It is convenient to provide centralized configuration of the values list for multiple one-type attributes simultaneously.
  • Select a set from the list.
  • Click Add - From template.
  • Select an additional  attribute (data) from the list of the existing attributes (data). The type and all the values of the sample additional  attribute (data) will be copied to the new element. Fill in all required fields. By default, the attribute becomes common with the set of the used attribute. You can not enter new values. You are warned that the values list is shared with another attribute.

Adding common additional attributes (data)

  • To use this option, select the Common attributes and data in the Administration - General settings.
  • A common attribute is included in multiple sets. This variant is useful when both values list and the attribute must be common. IN lists and reports for items of different catalogs and documents, the common attributes are grouped and displayed as one attribute in one column or field. For example, you can use the common attribute Color to pick out all yellow shoes and clothes.
  • Select a set from the list.
  • Click Add - Common.
  • Select an existing common additional attribute (data). If the list does not include the necessary attribute, click Create. Fill in the required fields.
  • To add a common attribute to multiple sets, repeat the procedure for each set.

More about selecting additional attributes (data)

  • To delete a common additional attribute (data), delete it from all the sets that included the given attribute. To delete the attribute (data) permanently, you may have to delete the additional attributes (data).
  • When you mark the property for deletion in the set, the property value is saved in objects until they are cleared. The property marked for deletion in the set will be shown in the object window or the additional data window in a special way.
  • If necessary, you may clear the deletion mark.
