Description of access groups profile properties.

In the application there are predefined profiles for access groups. IN some cases predefined access groups profiles are not enough to configure required settings for the user access permission. IN such cases the administrator has an option to add new profiles of access groups.

As a rule, access groups profile combines several roles and also contains description of access kinds. IN terms of access kinds the administrator can customize the rights using access groups. Therefore, a profile is a certain template based on which the administrator manages access groups.

One or several access groups can be related to one profile. For example, based on the Accountant profile Head Office Accountants and Company Division Accountants in N-city access groups that differ from each other by permitted companies: "Head office" and "Company Division in N-city" can be configured.

Entering the access groups profile

  • Enter the Name of profile.
  • If necessary, it is possible to move access groups profile to the folder by filling in the Group (folder) field. Access group profiles can be grouped based on various features. Select the required folder from the list.
  • Select Permitted actions (roles) on the relevant tab.
  • Specify Access Restrictions on the relevant tab.
  • At the Note tab you can specify an additional information of access groups profile. Predefined access profiles also have a Description tab with detailed description of this profile and its purpose.

Permitted actions (roles)

  • When adding a new profile for the access groups, first it is necessary to define the content of the roles based on the needs of the target group of users on the Allowed actions (roles) tab.
  • Role specifies a set of actions over the application objects that the user can perform. Roles may correspond either to various work-related duties (or activity kinds) of the application users or to small-scale functions. For example, the Accountant role provides an access to the functional block necessary to perform position duties of an accountant, and to add an option to view files it is possible to appoint the Read files role. As a rule, roles either authorize view or provide an option to edit various data in the application.
  • Using check boxes select one or several roles.
    • Using button you can select all roles in the list
    • Using button you can cancel selection of all roles
  • It is possible to display Only selected roles using the corresponding button.
  • Using By subsystems button it is possible to display the list of roles:
    • alphabetically
    • groupped by subsystems

Access restrictions

  • On Access restrictions tab you need to define the content of access types that shall be specified in the profile.
  • Access kind is meant to be a rule authorizing access to the application data. Using the kinds you can set boundaries more precisely for the data area which is permitted for the participants of various access groups to work with.
  • In the simplest case if no access kind is specified in the profile, data access permission is specified only by the roles. For example if the profile contains the role for reading  Goods Receipt documents, then participants of access groups get an option to view all the documents. But if a Company access kind is added to the profile in addition to this roles, then you can permit the users to work with documents of specific companies; or vice versa, hide from the users the documents of some companies.
  • Therefore if the list of the roles in the profile answers the question "what must be available and what must not be available" (what functions and application data), then access kinds specify the way they shall be available (by what permitted and prohibited values).
  • It is recommended to define the content of access kinds in the profile taking into consideration the flexibility of access permission settings.
  • Specify the Access kinds.
    • Click Add, select from the list one of the access kinds.
    • You can delete unnecessary records using the corresponding command of More menu or shortcut menu by clicking the right button of the mouse.
  • You can select one of four Access values for each access kind specified in the profile:
    • All prohibited, exceptions are indicated in access groups - it is applied when access to all data shall be denied by default, but permissions can be set for individual objects in access groups.
    • All permitted, exceptions are specified in access groups - similar to previous version, but all data are permitted by default, and exceptions - deny can be set for individual objects in access groups .
    • All prohibited, exceptions are specified in the profile - it is used when an option to configure an access kind is not provided in access groups; i.e. all settings shall be completed directly in the profile and can not be changed in access groups. Such access kinds are hidden in access groups.
      • In this case you can enter exceptions.
      • Click Add, select values from the relevant lists.
      • You can delete excess values using the corresponding command of More menu or shortcut menu clicking right button of the mouse.
    • Permit all, exceptions are specified in the profile - similar to previous version.

Viewing unused access kinds

  • Using More - Display unused access kinds command you can view those access kinds that are not used at this moment (for example, this option is disabled in the application).

Viewing Access groups

  • For quick switch to the list of access groups created based on this profile, you can use the Access groups command of navigation panel.
