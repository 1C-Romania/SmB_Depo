Designed to manage the profiles in the access groups.

There may be multiple access group profiles with alike purposes, so you can create folders to simplify search of a necessary profile and a better understanding of the formation principles.

To open in the list, use the Access group profiles command in Administration - Configuring users and rights.

List filters

  • Use the Show field to display:
    • All profiles (by default)
    • Delivered - profiles from the application delivery
    • Not delivered - profiles not included in the application delivery
    • Outdated

Primary and secondary access group profiles

  • There are primary and secondary access group profiles:
    • The primary profile describes a set of access rights enough to perform specific tasks in the application.
    • Use secondary profiles to grant certain service rights to the user in addition to their primary profile. For example, besides the primary profiles Accountant and Storekeeper, there are such profiles as Corresponding with counterparties and Printing unposted documents.

Recommendations on creating primary access group profiles

  • In the primary profiles, it is important to provide a set of roles that will not allow unwanted access to the application data and functions and will be enough for users to work with their duties.
  • In particular, the primary profile should involve a number of secondary roles which are not directly related to the users' main activities, but still necessary. 
  • For example, apart from the role for adding and editing these documents, the operator entering documents needs the following roles:
    • Role for logging in the application with a thick, thin or web-client.
    • Predefined roles of Basic rights and Basic rights UT.
    • Roles for viewing the catalogs which items are to be selected in the document fields.
    • Roles for viewing the reports on document transfer, etc.
  • For example, the Auditor profile should include not only the primary role Reading individuals allowing viewing data about individuals, but also a number of secondary roles allowing basic options for working in the application (e.g. Basic rights, Starting thin client, etc.)

Entering access group profile

  • Click Create and fill in the necessary fields.

Adding access group profiles folder

  • Click Create group and fill in the necessary fields.
