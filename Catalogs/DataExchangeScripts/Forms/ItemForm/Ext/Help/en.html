The main purpose of the script is the automatic data synchronization according to the specified schedule. The script is required for schedule and data submission and acquisition order setup.

It is not recommended to import or export the data only even though the script allows doing this. A full data synchronization is recommended for one script.

Setting the synchronization scrypt

  • Enter a script Name. In order to make all users understand the purpose of this script, specify the names for databases involved into synchronization and application revision.
  • Configure the schedule for running the data synchronization script.
    • Select the Automatically on schedule check box. After this, the link (by default, the link is Schedule is not set) becomes available for selecting. Click the link to set a schedule. After the schedule confirmation, the link shows the schedule configured.
  • Select the infobases that will have data synchronization script (as a rule the script is configured for the exchange with one infobase and includes import and export of data) enabled. For each infobase select the connection method and action (import or export). Click Add, fill in the the relevant columns:
    • Infobase - Select data type. Then select infobase from the list of relevant type;
    • Connection - select connection method from the list (depends on the properties of the infobase selected);
    • Action - click thebutton to select Import data or Export data;
  • Write a Comment - a brief script description.

Running the script

  • Click Run the script. Attempt to synchronize data. The actions specified in the script will be executed consistently. As soon as the data synchronization is completed, all unsuccessful import or export of data will be marked with icon in the Infobase column.
