It is used to view the personal data access history registered by the application.

You can opene it by running the Personal data protection command in the Administration - Support and service section.

According to the Federal law of 06/27/2006 No. 152-FZ "About personal data" and subordinate legislation the application implements registration of the events as follows:

  • running the application
  • exit from the application
  • authentication
  • authentication denial
  • access to personal data
  • access denial to personal data

The application can register personal data access history with respect to a subject. To enable this, it is necessary to configure the registration of personal data access history.
Almost all of the events (except personal data access history) are registered by the application by default. To register the event of access to personal data of individuals, the administrator of the application should run the Support and service - Personal data access history command of the Administration section. Then, it is necessary to select the Personal data check box for registration of access to personal data, to select the Passport data check box for registration of access to passport data. After that, the application automatically registers the event of access to these data in the event log.

The administrator opens the event log, set the filter for the period and event, receives the list of the events registered by the application. The generated list can be printed out by standard means of the application.

The events related to the protection of personal data are fixed in the application automatically. The events registered by the application can be reviewed in the list.

Each event is fixed in a separate line of the list. Important events (considering the access to personal data) are marked in the list. The list displays:

  • Date, time - besides date and time, event is displayed by an icon of the event type;
  • Event - name of the event;
  • User related to the event;
  • Event info - additional information on the event (current user of the OS who logged on into the application, application catalog linked to personal data into which the user logged on etc.)

Viewing the event

  • To view the detailed information on the event, double-click the right button of a mouse or click the Viewing the current event button.
  • You can also use the command of a context menu by clicking the right button of a mouse, or on the More menu.

Events filtering

  • Not all, but only some events are shown in the list according the number which specified in the Number of selected events field.
  • By clicking the Set date interval for viewing button you can change the interval of events shown, you can also double-click on the content of the Date, time column.
  • By clicking the Set filter by value in current column and Clear filter buttons, you can change the filter of events.
    • To do this, mark the event you need in the list (for example, Access), and then click the Set filter by value in current column button.
  • You can also use the commands of a context menu by clicking the right button of a mouse, or on the More menu.
