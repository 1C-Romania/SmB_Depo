It enables you to create an infobase backup.

It is recommended to back up data before any actions that can irreversibly change large amounts of data, for example, before bulk edit of the application object attributes. IN this case, you need to back up data manually.

To open it, in the Administration - Support - Backup section, run the Create backup command.

Creating backup

  • In the upper part of the window, you can see the last data backup date.
  • In the Backup directory field (required field), click Select, and then select a folder you want to save the infobase copy into. It is recommended to store a backup file on another computer or an external storage device.
  • If there are any current sessions, a warning message appears. Click the hyperlink to View all active sessions. Users have to close the application.
  • Once you set the required parameters, click Save backup.

Backing up

  • The application will try to back up data at once. The following message appears: "Initializing data backup." IN the window, you can see the Active users hyperlink. Click the hyperlink to see a list of user sessions. The application tries to end all user sessions.
  • Once all user sessions are ended, the application locks data and creates a backup.
  • If the application fails to end all user sessions, it will prompt you to back up data with the current user sessions. Administrator can continue if the application users are not entering any data (for example if users forgot to log out at the end of the day). Otherwise, end user sessions on their computers.
  • Once the backup is successfully completed, the application opens again. The backup completion message appears. It contains a backup file path.
  • If the application fails to forcibly end sessions, and administrator does not take any actions, the data may not be backed up then. A warning message appears informing you that the data is not backed up, and showing a list of sessions that failed to end (for example, Designer).

Run the application if any errors occurred during backup

  • Before you start backing up data, a message appears. It informs you how to start the application.
  • To log on into the database if the backup failed, run 1C:Enterprise with the following parameters: ENTERPRISE /F <database path> /CAllowUserAccess /UC Backup.
