There are two methods of a database update that can be applied in addition to each other:

  • using exclusive data processors:
    • users operation is locked.
    • it is applied for updating those data that shall be processed by the moment of users logon to the application.
      • the less such data, the less downtime is.
  • using delayed data processors:
    • users are allowed to log on to the application.
    • it is applied for updating the "archived" part of the database which is rarely used by users or which allows users to start their work not waiting the end of data processing.

Database update

  • After application upgrade (in the Designer mode or using basic versions of the Configuration update assistant) it is required initially to run the application with Administrator rights. The application will complete final update procedures at the start, this process may take a long time (depends on the update and computer performance).
  • Before the application initial start, it is recommended to close all user sessions. It is necessary for correct update of the application version.
  • In case users are still working in application it is possible to close active sessions preventing the update in a file mode, and to lock application logon temporarily (in client-server version the start is interrupted with a message informing about active sessions). By clicking the link you can view the active users list and request them to stop their work, or to continue update of the database by clicking the Log out and repeat button.
  • After that, the active sessions of other users working in the application are prepared to log out. Application displays a message informing the users to complete their work in all active user sessions. Application operation is aborted within the time specified in the message and the update continues.
  • Still the application is running and an empty desktop with a Application version Update message is launched. Message displays the number of new database version after update. Progress bar indicates the database update progress - graphically and in percent. To get detailed information about the database update process follow the Technical update info link from the events log.
  • In case the outdated platform version is launched, the application displays the message requesting platform version update and locks the desktop.
  • When a database is updated the application restarts automatically. The desktop in launched at once, but application operation is locked, and in order to continue operation it will be necessary to confirm the update authorization. The user is allowed to start working in the application after confirmation.
  • In case of update exception, the application displays an error message and it is necessary to contact the technical support. It is possible to view messages occurred during the update process by clicking the For more information see the events log link. IN this case, you can not continue work in the application.
  • In case of the successful update of the application, the What's new window appears. All application users will be unlocked, and they can log on to the application.

Descripton of application changes

  • After successful update of the application you can view the description of changes and improvements made in a new version (What's new in configuration window), go to the events log to read technical events occurred during the update process, and also, run and then monitor the delayed data processing progress (in case it is provided in this version of the application). Later, you can opent the What's new window from the Main section.

Aspects and advantages of delayed database update

  • In client-server mode the application update can provide for additional data processing, which runs in the background mode in order to reduce downtime. At that, users can start their work in a application, not waiting for the completion of such processing. During the processing some of the application functions can be temporarily unavailable as the update is not yet completed.
  • In this case, the documents that are not yet processed will be unavailable. At the attempt to open such a document, the application displays a message informing that document processing is temporarily prohibited, as the application update was not completed. Some reports can also be unavailable. At the attempt to generate a report, the application displays a message informing that the report can contain incorrect data as the application update is not completed.

Delayed update process control

  • In those cases, when application update provides for the background delayed data processing the administrator can review delayed application update procedures progress by running the Update results and data processing command in the Administration - Support and maintenance section, or by clicking the Additional procedures in progress link in the What's new in configuration window.
