Designed to estimate the application performance.

You can use the embedded APDEX data collection/analysis recognized as the international standard for application performance estimation.

To estimate performance, fill in the Key operations list before you start working.

To estimate performance with APDEX, take the following steps:

  • Find the priority of each key operation of the application.
  • Find the target performance time for each key operation.
  • Collect information about the actual performance time of each key operation.
  • On the basis of the collected data, define the performance level and interpret the acquired values.

To view and analyze the performance estimations, use the Performance scores command in the Administration - Support and service section. 

Table

The performance scores are added to the table including the following columns:

  • Key operation - a name of the key operation. The table shows all available Key operations in the relevant list. By default, the predefined Overall system performance key operation is shown only in the table.
    • Use the  Add and   Delete buttons  to create a list of key application operations for estimating the application performance.
    • Whenever a key operation is added to the table, you can not select it in the list (i.e. one operation can be added to the table just once). 
  • Priority - it is displayed from the Key operations list.
    • Use the  Up and   Down buttons to move key operations in the list.
    • When moving a key operation in the list, you automatically change its Priority.
    • The top and bottom operation is of the highest and lowest priority respectively. Therefore, you can set up the priority of each operation by moving them in the list.
    • The Overall system performance entry is always at the top; it has no priority and displays the application performance summary.
  • Target time (in seconds) - it is displayed from the Key operations list. The performance level will be related to this value. If the target time values change, the performance values will change, too. 

Setting period

  • Before you estimate performance, specify the period to observe.
    • Enter the Start date and End date. Use the  button to select a calendar date and change time if necessary (by default, 0:00:00).
  • In the Step field, you can set a number of results displayed in the period. For example if the period is one day and the step duration is one hour, the window will show 24 results (for each key operation). The per-hour result will be calculated by the special APDEX formula. If no operations were executed within one hour, the relevant table cell will show "0". The estimations will be made during the specified key operations, e.g. when posting a document.
  • The estimation results will be shown in the Performance level window in the Table and Diagram tabs.

Updating table

  • The list is not dynamic. To update the diagram and table data after editing the Period and/or Step fields, you can use the Update performance scores command.

Calculating target time

  • Select a necessary operation (except for Overall system performance) and execute the Calculate target time from APDEX command from the context menu. You can not execute the command if time has been measured for this key operation. 

Performance level

  • The performance level value is between 0 and 1. According to the level value, its field is shown in a specific color. The level can be understood as follows:

Filtering scores

To filter your performance scores, click  Filter and use a radio button to select one of the following:

  • The Filter - Better or Worse field
  • To compare, select an APDEX level: Good, Fair, Poor
  • The Condition field automatically shows the APDEX score corresponding with the level
  • To start filtering, click OK

Key operation execution history

  • To view the Key operation execution history, double-click the table score. You can see a period of estimations stated in the table, the key operation parameters filled in from the key operation list, and the number of estimations. The estimation table shows information about each estimation: Duration, End time, User.

Diagram

  • For a better convenience, the Diagram page shows the table scores as graphs, where each graph corresponds to a key operation.

Exporting performance estimation

  • You can automatically export your performance scores to external files in background. For example, you can analyze performance in another application.
  • To set up exporting of this data, click More - Configuration. Generated external files and the files containing the settings corresponding with the acquired estimations are packed in a ZIP archive.
  • To export performance estimations manually, click More - Export data.
