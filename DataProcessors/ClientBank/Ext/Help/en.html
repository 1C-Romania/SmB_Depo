Client bank

Use the Client bank data processor to transfer payment orders to the bank and receive bank statements using direct connection to the bank service or via a special client-bank application.

Customizing client-bank exchange

You can customize your client-bank exchange from the data processor form for each bank account of the enterprise organizations. If the organization has multiple bank accounts, you will have to define more settings. When you select a bank account, the relevant setting is used automatically. To maintain direct exchange with the bank, create an EDF setting for your bank and switch it to Active.

Supported scenarios

Use the Client-bank data processor to:

  • Send payment orders generated in the infobase to the bank via electronic communication channels.
  • Import information about transactions with the organization bank accounts.

According to the configured client-bank variant, you can directly contact the bank to send payment orders and receive bank statements without using external applications or use a client-bank application. If the bank in a selected account supports direct exchange, you will use direct exchange. If direct exchange is not selected or the account is not specified, you will use a client-bank application.

The data exchange between the configuration and the client-bank application uses an intermediary data storage, an exchange file of the certain format.

To send payment documents prepared in the configuration, you need to export them to the exchange file. The client-bank application reads and processes this file. The result is transferred to the bank via telecommunication channel.

To receive bank statements, the client-bank application initiates data receipt from the bank and generates an exchange file. After that, this file is read and processed.

Required condition for interaction with a client-bank application: the bank application must support the 1C:Enterprise - Client-bank exchange standard.
