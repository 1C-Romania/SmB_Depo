This panel is used to configure data synchronization for the infobase.

  • Directory of temporary Windows files
    		•  Path to a single directory of temporary files of all servers in the cluster used for servers running under Windows OS.
  • Directory of temporary Linux files
    		•  Path to a single directory of temporary files of all servers in the cluster used for servers running under Linux operating system.

    You may leave the directories of temporary files empty if only one server is running in 1C:Enterprise cluster, or the application works in the file mode.

    If all cluster servers are running under operating systems of one family (Windows/Linux), then it is not required to fill in the second field.

    The directory must be available to read and write for all cluster servers, including, for all users of operating systems whose names are used to run 1C:Enterprise server. UNC path shall also be the same for all servers of one family - for this you shall connect the directory to servers under one name.

    In the Directory of temporary Linux files field you can specify only local path, for example, /home/path.

    To use UNC path, it is necessary to make a reference to it. To do it, do the following using administrator rights:

    mkdir folder
    --- enter directory name instead of folder, for example, in /home directory do mkdir shared - it creates a shared directory with full path /home/shared.

    mount -t cifs //server/share folder -o //domain/user, password=pwd

    - where //server/share - UNC path to another computer, the folder - path of the directory on a Linux machine, for example \home\shared, //domain/user - domain/user name if needed, pwd - user password (if needed).

    If the name and password are not necessary, the command is as follows:
    mount -t cifs //server/share folder

    After this specify the folder, for example, /home/shared

    For more details on 1C:Enterprise server administration see documents of 1C:Enterprise platform.
