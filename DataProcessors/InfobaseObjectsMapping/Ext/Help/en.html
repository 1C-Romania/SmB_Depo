Designed to map data of two applications.

To open the list, use the Map command on the Data mapping page of the Interactive data synchronization wizard. You can also double-click the line with data-mapping problems. The list consists of two columns corresponding with the exchange infobases. The data is grouped by application objects (documents and lists). At the list bottom, there is a status bar showing the number of mapped and unmapped items.

In the Configure display field, you can select data to display in the list. By default, it is Unmapped data.

Mapping objects

  • Click Map automatically (recommended) and select check boxes for necessary fields. Some fields are selected by default. To confirm your selection, click Map. After search, the mapped data is displayed. Click Apply to confirm.
  • After automatic mapping you can map the remaining objects manually or edit your mapping. Select the necessary objects of the two bases and click Unmap to try to map the objects manually. Then click Map.
  • To confirm, click Save and close.

Configuring mapping table fields

  • Click Columns to add list fields and columns. Select check boxes for additional fields if necessary and click Apply to confirm.

Receiving data from another application

  • To receive data of another application, click More -  Export data from another application.

Object mapping sequence

  • It is recommended to map and load data considering its links. Especially when the field is used to map objects.
  • For example, a configuration has a counterparty contract catalog subordinate to a counterparty catalog. The counterparty contracts are mapped according to the owner catalog, i.e. the counterparty catalog. Consequently, to map the data correctly, you need to firstly map and import the counterparty catalog and then map and import the counterparty contract catalog.
  • Otherwise, the mapping-table fields may contain empty links like:
    <Object not found> (26:a0b9001b24e002fe11dfba347dd41412).
  • An empty link refers to an object which has not been exported yet from the exchange message to the current infobase.
