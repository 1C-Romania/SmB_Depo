It enables you to register data changes in the application. It is required to sync data with other applications.

To open the list, in the Data sync list, run the More - Sent data command.

Message numbers are registered. Click a hyperlink with message numbers to view or edit them.

The list consists of two parts:

  • The left part consists of two columns:
    • Configuration object - the application objects (catalogs, documents, etc.) listed as a tree.
    • Quantity - number of registered and changed (or newly entered) / not imported items in these objects.
  • The right part changes depending on the object you select in the left part:
    • In the Details column, you can see a detailed description of the left part statistics of the selected object group.
    • The right part is divided into two columns on the selected object:
      • Name of a particular object (for example, Currency) - a list of all changed and not imported list items.
      • Number of message sent - message number that contains changed list items (message number or Not imported is specified.)

Filtering changes

  • In the Show field, select the value you need from the list:
    • All
    • Imported only
    • Not imported only

Registering changes and canceling registration

  • The application automatically registers edited and new objects. To manually register the objects, select an object in the tree using check boxes, and then click Register:
    • All objects of the selected types - you can select an object and all its child objects
    • Objects of the selected type with autoregistration
    • Single object (select from the list or create a new one)
    • Filtered objects (set a filter, filter the objects you need)
    • Request results
  • To cancel the data sync to other applications, in the object tree, click a root (an infobase), and then click Cancel registration - All objects of selected type. All settings are similar to the previous ones.

Settings

  • To configure the object changes registration parameters, click More - Settings. You can specify autoregistration parameters, set a registration message number, and specify a path to the query console directory if required.
  • Use the check boxes to enable Autoregistration of:
    • Migrations with documents
    • Sequences with documents
  • Use option buttons to customize Setting message numbers while registering:
    • Register as not sent
    • Use message number of the exchange node
  • To use the query console, click In directory, and then specify a path to the query console. If you do not need it, click Do not use.
  • Select the check box to apply the Standard subsystems library while registering the object changes of data migration limits.
  • To apply the settings, click Write.

      Viewing import result for distributed infobase

      • To view the import result, in the left part of the list, click the object with registered changes. IN the right part of the list, an item list appears. Click the item you need, and then click Show export result or More - Standard export result.
