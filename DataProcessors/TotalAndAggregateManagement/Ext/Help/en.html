Totals control includes procedures related to the accumulation and accounting registers totals as well as to the aggregates of the turnover registers of accumulation. Totals control is divided into two modes :

  • Commonly used opportunies - allow by "one touch" to perform the most common actions that occur during the system maintenance.
  • Full opportunities - allows to perform all operations as with a single register so with an arbitrary list of registers

    To switch modes there is a hyperlink in the bottom right corner of the form. Form remembers the mode of opening .

    Commonly used opportunies

    Operations with totals

    • Set the period of the calculated totals - to speed up the system  the application Sets the period of the calculated totals for all accumulation and accounting registers with enabled totals. Registers for accumulation - at the end of the previous month, registers for accounting - at the end of the current month.
    • Enable the use of totals - enables the use of totals for all registers which have this mode disabled (e.g. as a result of an emergency termination of any procedure that disables the use of the totals).

    Operations with aggregates

    • Rebuild and fill in - for all the turnover registers of accumulation for which the aggregates mode is enabled and their use is Seted, the application performs an operation of rebuilding of the aggregates use and performs the filling in of the aggregates. Operation can take a long time.
    • Get the optimal aggergates - receives a list of the optimal aggregates for all the turnover registers of accumulation for which in the Designer IS set the aggregates. Will be required  folder where will be saved the xml-files with the lists of optimal aggregates. Each file will be named by identifier of the corresponding register. So, for the register Sales the file  will be named as Sales.xml

    Full opportunities

    In the full opportunities mode it is available a multiple selection mode. Any action applies to all the selected objects or to that object which is pointed by the cursor.

    Totals

    On this tab it is possible to perform the following operations:

    • enable and disable the use of totals,
    • enable and disable the use of current totals,
    • enable and disable the totals division,
    • setup of the selected period for the totals calculated,
    • perform the recalculation of the totals, current totals or perform the recalculation for the selected period.

    Only those registers are listed which for the current user have Seted the right of Totals management.

    Aggregates

    On this tab it is possible to perform the following operations:

    • set the mode of the register use: totals or aggregates,
    • enable and disable the use of aggregates,
    • enable the rebuilding of the aggregates use;
    • perform the aggregates update,
    • perform the aggregates clearing;
    • perform the calculation of the optimal aggregates list. Will be required the folder where the xml-files with the lists of optimal aggregates will be saved. Each file will be named by the identifier of the corresponding register. So, for the register Sales the file will be named as Sales.xml. After the operation is performed in the registers list will be highlighted in bold those registers for which you want to update the list of aggregates for the calculated optimal aggregates. Update must be performed in the Designer.

    At the bottom of the form it is displayed the list of aggregates of the register which is pointed by the cursor in the upper list. The list includes only those turnover accumulation registers which have set for the current user the right of  Totals management and set aggregates in the Designer.
