Designed to set up infobase backup.

Sometimes, you lose data, e.g. because of a power failure, computer bug or incorrect operations. To mitigate the risk of data loss, the application administrator should regularly create an infobase backup to restore a functional data version when necessary. The backup frequency depends on how often new data is added. Backup is recommended before any operations related to modifying much data, e.g. before modifying a group of objects.

Any time, you can restore data from a previously created backup. However, you only restore the data entered before the backup was created. All data entered later will be permanently lost. The backup schedule should minimize data loss.

In 1C:Enterprise, backup is available only for the file operation variant. For the client server variant, it is recommended to create backups with DBMS. For more information, see the "1C:Enterprise 8.2. Client Server Variant. Administrator Guide" book.

Open the dialog box with the relevant link in the warning displayed when the application starts without backup configured. The warning prompts you to set up infobase backup. If ignored, the backup warning will appear every hour.

You can also use the Backup settings command in the Administration - Support and Service - Backup and recovery section. Below this command, the backup status is displayed. 

Backup settings (file mode)

  • In 1C:Enterprise, you can back up data either automatically or manually. To configure the automatic mode:

  • Select the Back up automatically check box.

  • Switch to one of the options:
    • On schedule - back up automatically. Follow the link to set the backup schedule. The link displays the entered schedule.
      • Note: if a backup session starts when the application is not working, backup will fail.
    • When logging off - every time the administrator logs off, a dialog box prompting to back up appears. To start a backup session, confirm your selection by clicking Finish. In the Log off window, the Create backup check box is selected by default. If you do not want to see this dialog box again, clear the check box.
  • To select Backup directory, click  Select, specify the path to the folder for backups, e.g. E:\Backup, in the special required field. It is recommended to save your backup on another computer or external media.
  • Switch to the necessary Backup retention variant to specify the retention period or the number of the copies to retain: 
    • All - you can select this variant if the hard drive of the backup directory has enough space.
    • Last 6 months (by default). You can limit the backup retention period.
    • Last 10 copies (by default). You can limit the number of backups.
  • Click Ready to confirm your selection and return to the application. The application will switch to automatic backup.

Backup with external tools

  • You can cancel automatic backup if you back up files with external tools, e.g. DBMS for client server operation, or external backup applications. 
  • To do it, clear the Back up automatically check box.

Backup for files in volumes

  • To save files in volumes on the drive, these files also require backup with other tools.

Backup errors

If a backup session failed and the temporary log file has the text: "An exception was thrown in createConnection: Error, Automation server can't create object", you are probably using a 32-bit comcntr.dll component on a computer under a 64-bit operating system. To resolve this issue, use the 64-bit comcntr.dll component from the 1C:Enterprise x64 Server delivery and register it by entering into the command line:

regsvr32 <64-bit comcntr.dll directory>\comcntr.dll

Important: to enter the infobase if a backup session failed, start 1C:Enterprise with the following parameters: ENTERPRISE /F <path to infobase> /AllowUsersWork/UCBackup.
